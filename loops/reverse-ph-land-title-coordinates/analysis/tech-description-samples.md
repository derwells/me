**Survey plan:** PLS-1110 (Public Land Subdivision)

## Cross-Sample Analysis: Format Variation Inventory

### A. Tie Point Reference Types
| Code | Expansion | Example |
|------|-----------|---------|
| BLLM | Bureau of Lands Location Monument | "BLLM No. 1, PLS-1110" |
| B.L.L.M. | Same (with periods) | "B.L.L.M. 1, municipality of Malabon" |
| BLLM. | Same (trailing period) | "BLLM. No. 1, Pls-793" |
| B.B2. | Barangay Boundary Monument #2 | "B.B2. No. 42, Angeles Cadastro" |
| BLBM | Bureau of Lands Barrio Monument | "BLBM No. 1, Makar" |
| L.W. | Lot Witness (cadastral block monument) | "L.W. 22, Piedad Estate" |
| B.L. Mon. | Bureau of Lands Monument (general) | "B.L. Mon." (boundary reference) |

**Parser implication:** Tie point extraction must handle multiple monument code formats, both with and without periods, and with varied descriptors (municipality, cadastral project, landmark name).

### B. Bearing Notation Variants
| Variant | Example | Notes |
|---------|---------|-------|
| Degree symbol | "S. 65° 02' E." | Most common in older typewritten titles |
| Spelled-out deg | "N. 11 deg. 05' W." | Common alternative |
| No minutes | "N. 51° W." | Degrees only; minutes absent |
| Due cardinal | "Due North" | No angle at all; purely N/S/E/W |
| Missing N/S prefix | "50 deg. 50' E." | Transcription/OCR error variant |
| Missing E/W suffix | "N. 22 deg. 40'" | Transcription/OCR error variant |
| No periods | "N 77° 42' W" | Some typefaces/fonts omit periods |

**Parser implication:** Bearing parser must tolerate: (1) ° vs "deg.", (2) missing minutes, (3) "Due North/South/East/West" as special case, (4) missing quadrant prefix/suffix as error condition.

### C. Corner Sequence Phrasing
| Variant | Example |
|---------|---------|
| Standard | "thence N. 77° 42' W., 16.41 m. to point 2" |
| Quoted point | "thence S. 9° 35' E., 27.99 m. to point "2"" |
| Implied thence | "S. 36 deg. 43' E., 98.73 m. to point 2" |
| No point number | "to the point of beginning" |
| Named point | "to point of beginning" (no "the") |
| Numbered close | "to point 1, point of beginning" |

### D. Area Statement Formats
| Variant | Example |
|---------|---------|
| Words + numerals | "FOUR HUNDRED EIGHTY FIVE (485) SQUARE METERS" |
| Title case | "One Thousand Twenty-Four Square Meters (1,024 m²)" |

| With "only" | "32,156 square meters, only" |
| With "more or less" | "500 SQUARE METERS more or less" |

## Structural Template (Canonical Technical Description)

Based on all samples, the canonical structure is:

```
[Lot identification line]
A parcel of land (Lot [N], [Block B,] of [survey plan prefix-number], [being a portion of {parent lot}],
[L.R.C. {record ref}]), situated in [Barangay], [Municipality/City], [Province], Island of [Luzon/Visayas/Mindanao].

[Boundary description — optional]
Bounded on the [N.] along line [1-2] by [description]; on the [E.] ... [etc.]

[Tie line + traverse]
Beginning at a point marked "[1]" on plan, being [BEARING], [DISTANCE] m. from [TIE POINT REFERENCE],
thence [BEARING], [DISTANCE] m. to point [2];
thence [BEARING], [DISTANCE] m. to point [3];

[Footer]
All points referred to are indicated on the plan and [are] marked on the ground [with {monument type}].

[Approved {DATE}; registered in accordance with ...]
```

## Key Findings for Parser Design

1. **Tie point parsing requires regex for all monument code variants** (BLLM, B.L.L.M., BLLM., BLBM, B.B2., L.W., B.L. Mon.) plus the descriptive suffix.

2. **Bearing parser must handle 5 formats**: (1) `N DD° MM' E`, (2) `N DD deg. MM' E`, (3) `N DD° E` (no minutes), (4) `Due North/South/East/West`, (5) missing prefix/suffix (error recovery).

3. **Corner calls are terminated by**: "to point N", "to point "N"", "to the point of beginning", "to point of beginning", "to point N, point of beginning".

4. **Area is always the last substantive data point** before the footer. It can be in all-caps, title case, or lowercase. Numerals appear in parentheses or after a comma.

5. **Datum is NOT explicit in the body text** — must be inferred from survey date (pre/post 1993) and any explicit notation on the plan cover (not in the TD text itself).

6. **Survey plan number prefix** carries metadata: region code (02, 03, 04...), survey type (Psd, Csd, Psu, Cad), and sometimes suffix letter.

7. **"Grid" bearings** in pre-1993 surveys = old PTM zone north. Zone must be inferred from the property location (province → PTM zone → PRS92 zone mapping). See Wave 2 prs92-datum-parameters analysis.
