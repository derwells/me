# P11 — Rent Roll Preparation

*Feature Spec | Depends on: F0, P1, P5, P6, P7 | Implementation order: 5 of 15 (MVP pipeline: F0→P1→P5→P6→P11)*

---

## Summary

Generate the monthly 26-column rent roll — the central report delivered to the external accountant for BIR filings — and the semi-annual LIS (Lessee Information Statement) per RR 12-2011. P11 is a pure aggregation feature: no new business logic computations, no mutations to upstream data. It queries data from F0 (leases, tenants, rentables), P1 (current rent periods), P5 (charges, invoices), P6 (payments, balances, Form 2307), and P7 (security deposits) to assemble per-unit rows with cross-reference flags for downstream BIR forms (2550Q, 1702Q, SAWT, LIS). Generated reports are snapshot-persisted as JSONB for reproducibility and export. Both roles (Admin and Accountant) can view and export — this is the primary accountant deliverable.

### 1.3 JSONB Schema — `report_data` (Rent Roll)

Each element in the `report_data` array is a row object with 26 fields corresponding to the rent roll columns:

```ts
// packages/db/src/schema/reporting.ts (Zod schema paired with JSONB column)
import { z } from "zod";

export const rentRollRowSchema = z.object({
  // Identity (cols 1-7)
  unitReference: z.string(),                         // Col 1: "3B" or "G1"
  unitType: z.enum(["RESIDENTIAL", "COMMERCIAL"]),   // Col 2
  tenantName: z.string(),                            // Col 3: billing_name or full_name
  tenantTin: z.string().nullable(),                  // Col 4
  leaseStart: z.string(),                            // Col 5: ISO date
  leaseEnd: z.string().nullable(),                   // Col 6: ISO date or null (m-t-m)
  leasedAreaSqm: z.string().nullable(),              // Col 7: numeric as string

  // Billing (cols 8-15)
  baseRent: z.string(),                              // Col 8: pre-VAT monthly rent
  vatStatus: z.enum(["VATABLE", "EXEMPT"]),          // Col 9
  vatAmount: z.string(),                             // Col 10: VAT billed this period
  totalBilled: z.string(),                           // Col 11: all charges sum (incl. VAT)
  amountCollected: z.string(),                       // Col 12: cash received this period
  ewtWithheld: z.string(),                           // Col 13: EWT deducted by corporate
  netCollected: z.string(),                          // Col 14: cash + EWT

  // Balance (cols 15-16)
  priorBalance: z.string(),                          // Col 15: balance before this period
  currentBalance: z.string(),                        // Col 16: balance after this period

  // Documents (cols 17-18)
  invoiceNumber: z.string().nullable(),              // Col 17
  receiptNumber: z.string().nullable(),              // Col 18: OR number from payment

  // 2307 tracking (col 19)
  form2307Status: z.enum([
    "N/A", "EXPECTED", "RECEIVED", "DELIVERED", "RECONCILED", "DISPUTED",
  ]),                                                // Col 19

  // Additional (cols 20-26)
  depositHeld: z.string(),                           // Col 20: security deposit amount
  leaseStatus: z.string(),                           // Col 21: ACTIVE, MONTH_TO_MONTH, etc.
  daysOverdue: z.number().int().min(0),              // Col 22: 0 if current
  notes: z.string(),                                 // Col 23: vacancy, holdover, disputes
  lisFlag: z.boolean(),                              // Col 24: row qualifies for LIS
  sawtFlag: z.boolean(),                             // Col 25: row has 2307 data for SAWT
  adjustmentFlag: z.boolean(),                       // Col 26: pro-rated, credit memo, etc.

  // Metadata (not visible columns — used for filtering/grouping)
  leaseId: z.number().int(),
  tenantId: z.number().int(),
  rentableId: z.number().int(),
  leaseRegime: z.string(),
});

export type RentRollRow = z.infer<typeof rentRollRowSchema>;
```

### 2.4 Procedure Details

**`rentRoll.generate` (mutation, adminProcedure):**

Transaction-wrapped:
1. Validate property exists
2. Check no FINALIZED rent roll exists for this property+period (CONFLICT error if exists; DRAFT can be overwritten)
3. Delete existing DRAFT for same property+period (overwrite)
4. Assemble rent roll rows (see §4.1 Rent Roll Assembly Logic)
5. Compute summary data (see §4.2 Summary Computation)
6. INSERT `rent_roll_report` with status=DRAFT
7. Log audit entry
8. Return created report

**`rentRoll.finalize` (mutation, adminProcedure):**

1. Load report by ID, verify status = DRAFT (PRECONDITION_FAILED if FINALIZED)
2. UPDATE status = 'FINALIZED', finalized_at = now()
3. Log audit entry
4. Return updated report

**`rentRoll.delete` (mutation, adminProcedure):**

1. Load report by ID, verify status = DRAFT (PRECONDITION_FAILED if FINALIZED)
2. DELETE report
3. Return success

**`rentRoll.getPreview` (query, protectedProcedure):**

Same assembly logic as `generate` but does NOT persist. Returns rows and summary directly. Used for the live preview before saving.

**`rentRoll.generateLIS` (mutation, adminProcedure):**

Transaction-wrapped:
1. Validate property exists
2. Parse semester into as_of_date (H1→Jun 30, H2→Dec 31) and filing_deadline (H1→Jul 15, H2→Jan 15 next year)
3. Delete existing LIS for same property+semester (overwrite)
4. Query active leases as of as_of_date
5. Assemble 9-column LIS rows (see §4.3 LIS Assembly Logic)
6. INSERT `lis_report`
7. Log audit entry
8. Return created report

**`rentRoll.markLISFiled` (mutation, adminProcedure):**

1. Load LIS by ID
2. UPDATE is_filed = true, filed_date = input.filedDate
3. Dismiss any COMPLIANCE alert for this LIS deadline
4. Log audit entry
5. Return updated report

**`rentRoll.getSAWTData` (query, protectedProcedure):**

1. Query `form_2307_record` WHERE quarter = input.quarter AND status IN ('RECEIVED', 'DELIVERED', 'RECONCILED')
2. JOIN tenant for TIN and name
3. Assemble SAWT rows with sequence numbering (see §4.4 SAWT Assembly)
4. Compute summary: total records, total tax_base, total tax_withheld
5. Return rows + summary

**Interactions:**
- **Property selector:** Dropdown filtered to properties the user has access to (single property for TSVJ MVP)
- **Period selector:** Month/year picker. Defaults to previous month (current month's data is incomplete)
- **Generate button:** Admin-only. Calls `rentRoll.generate`. If DRAFT exists for same period, shows ConfirmDialog ("Overwrite existing draft?"). If FINALIZED exists, shows error toast
- **Export buttons:** Both roles. Triggers `rentRoll.export` → client-side SheetJS generation → download. XLSX includes 3 sheets: Main (26 cols), VAT Summary, 2307 Summary
- **Finalize button:** Admin-only. ConfirmDialog ("Finalize this rent roll? It cannot be modified after finalization."). Calls `rentRoll.finalize`
- **Column visibility:** TanStack Table column visibility toggle for managing 26-column width
- **Row highlight:** Rows with `daysOverdue > 0` highlighted with warning color. Rows with `adjustmentFlag = true` marked with asterisk

## 4. Business Logic

### 4.1 Rent Roll Assembly Logic

The rent roll generation queries multiple tables and assembles one row per lease-rentable combination. No computation module needed — this is pure SQL aggregation + data assembly in the tRPC router.

```
FUNCTION assembleRentRollRows(propertyId, billingPeriod) → RentRollRow[]:

  periodStart = firstDayOf(billingPeriod)    // e.g., 2026-03-01
  periodEnd = lastDayOf(billingPeriod)       // e.g., 2026-03-31

  // 1. Get all leases active during this period
  //    Status IN (ACTIVE, MONTH_TO_MONTH, EXPIRED) with overlap check:
  //    lease.date_start <= periodEnd AND (lease.date_end IS NULL OR lease.date_end >= periodStart)
  //    PLUS holdover leases (if any)
  leases = query lease
    JOIN lease_rentable ON lease.id = lease_rentable.lease_id
    JOIN rentable ON lease_rentable.rentable_id = rentable.id
    JOIN room ON rentable.room_id = room.id
    JOIN tenant ON lease.tenant_id = tenant.id
    WHERE room.property_id = propertyId
      AND lease.status IN ('ACTIVE', 'MONTH_TO_MONTH', 'EXPIRED', 'HOLDOVER')
      AND lease.date_start <= periodEnd
      AND (lease.date_end IS NULL OR lease.date_end >= periodStart)
    ORDER BY room.name, rentable.name

  rows = []
  FOR EACH (lease, rentable, tenant) IN leases:

    // Col 1-7: Identity
    unitRef = formatUnitPath(room.name, rentable.name)  // e.g., "3B" or "A-102"
    unitType = rentable.unit_type                        // RESIDENTIAL or COMMERCIAL
    tenantName = tenant.billing_name ?? fullName(tenant)
    tenantTin = tenant.tin
    leaseStart = lease.date_start
    leaseEnd = lease.date_end  // null for month-to-month

    // Col 8: Base rent — current RecurringChargePeriod amount
    rentPeriod = query recurring_charge_period
      JOIN recurring_charge ON recurring_charge.id = recurring_charge_period.recurring_charge_id
      WHERE recurring_charge.lease_id = lease.id
        AND recurring_charge.is_active = true
        AND recurring_charge_period.period_start <= periodEnd
        AND (recurring_charge_period.period_end IS NULL OR recurring_charge_period.period_end >= periodStart)
      LIMIT 1
    baseRent = rentPeriod?.amount ?? "0.00"

    // Col 9: VAT status
    vatStatus = determineVATStatus(rentable.unit_type, baseRent, lease.lease_regime)
    // EXEMPT if residential ≤ ₱15K; VATABLE otherwise (uses @tsvj/computations determineVAT)

    // Col 10-11: Charges for this period
    charges = query charge
      WHERE charge.lease_id = lease.id
        AND charge.billing_period = billingPeriod
        AND charge.status IN ('FINALIZED', 'PAID', 'PARTIALLY_PAID')
    totalVat = SUM(charges.vat_amount)
    totalBilled = SUM(charges.amount)  // amount = base_amount + vat_amount

    // Col 12-14: Payments in this period
    payments = query payment
      JOIN payment_allocation ON payment.id = payment_allocation.payment_id
      JOIN charge ON payment_allocation.charge_id = charge.id
      WHERE charge.lease_id = lease.id
        AND payment.date_issued BETWEEN periodStart AND periodEnd
        AND payment.is_reversed = false
    amountCollected = SUM(payments.amount)   // Cash portion
    ewtWithheld = SUM(payments.ewt_withheld) // EWT deducted by corporate
    netCollected = amountCollected + ewtWithheld

    // Col 15-16: Balances
    tenantBal = query tenant_balance WHERE tenant_id = tenant.id
    currentBalance = tenantBal?.balance ?? "0.00"
    // Prior balance = currentBalance + totalBilled - netCollected
    // (balance as of start of period, before this period's charges and payments)
    priorBalance = Decimal(currentBalance).minus(totalBilled).plus(netCollected).toString()

    // Col 17-18: Document numbers
    invoice = query issued_document
      WHERE issued_document.lease_id = lease.id
        AND issued_document.billing_period = billingPeriod
        AND issued_document.document_type = 'INVOICE'
      ORDER BY issued_document.document_number ASC
    invoiceNumber = invoice[0]?.document_number  // First invoice for period
    receipt = query issued_document
      WHERE related to payments above AND document_type = 'RECEIPT'
    receiptNumber = receipt[0]?.document_number

    // Col 19: 2307 status
    IF tenant.is_corporate:
      quarterLabel = billingPeriodToQuarter(billingPeriod)  // "2026-03" → "2026-Q1"
      form2307 = query form_2307_record
        WHERE tenant_id = tenant.id AND quarter = quarterLabel
      form2307Status = form2307?.status ?? "EXPECTED"
    ELSE:
      form2307Status = "N/A"

    // Col 20: Security deposit
    deposit = query security_deposit
      WHERE security_deposit.lease_id = lease.id
        AND security_deposit.status IN ('HELD', 'PARTIAL_RETURN')
    depositHeld = deposit?.amount ?? "0.00"

    // Col 21-22: Lease status + days overdue
    leaseStatus = lease.status
    daysOverdue = 0
    IF tenantBal?.balance > 0:
      oldestUnpaid = query charge
        WHERE charge.lease_id = lease.id AND charge.status IN ('FINALIZED', 'PARTIALLY_PAID')
        ORDER BY charge.due_date ASC LIMIT 1
      IF oldestUnpaid:
        daysOverdue = MAX(0, daysBetween(oldestUnpaid.due_date, periodEnd))

    // Col 23: Notes
    notes = compileNotes(lease, rentable, charges)
    // "VACANT" for vacant units, "Month-to-month" for MTM, "Pro-rated" for mid-month,
    // "Holdover" for holdover, "Credit memo applied" if credit_memo exists, etc.

    // Col 24: LIS flag — row qualifies for semi-annual LIS
    lisFlag = lease.status IN ('ACTIVE', 'MONTH_TO_MONTH')

    // Col 25: SAWT flag — row has 2307 data for SAWT generation
    sawtFlag = tenant.is_corporate AND form2307Status IN ('RECEIVED', 'DELIVERED', 'RECONCILED')

    // Col 26: Adjustment flag — non-standard billing (pro-rated, credit memo, advance applied)
    adjustmentFlag = hasProRation OR hasCreditMemo OR hasAdvanceApplied

    rows.append(RentRollRow { ...all fields above... })

  // Add vacant unit rows
  vacantRentables = query rentable
    JOIN room ON rentable.room_id = room.id
    WHERE room.property_id = propertyId
      AND rentable.id NOT IN (SELECT rentable_id FROM lease_rentable
                              JOIN lease ON lease.id = lease_rentable.lease_id
                              WHERE lease.status IN ('ACTIVE', 'MONTH_TO_MONTH', 'EXPIRED', 'HOLDOVER'))
  FOR EACH vacantRentable:
    rows.append(RentRollRow {
      unitReference: formatUnitPath(room.name, vacantRentable.name),
      unitType: vacantRentable.unit_type,
      tenantName: "(VACANT)",
      // All monetary fields = "0.00", all nullable fields = null
      notes: "VACANT",
      lisFlag: false,     // Vacant units excluded from LIS
      sawtFlag: false,
      adjustmentFlag: false,
    })

  RETURN rows sorted by unitReference
```

### 4.2 Summary Computation

After assembling all rows, compute `summary_data`:

```
FUNCTION computeRentRollSummary(rows: RentRollRow[]) → RentRollSummary:

  occupiedRows = rows.filter(r => r.tenantName !== "(VACANT)")

  byUnitType = GROUP occupiedRows BY unitType:
    FOR EACH group: { unitType, count, totalBilled: SUM, totalCollected: SUM, totalBalance: SUM }

  vatSummary = {
    vatableSales: SUM(r.baseRent WHERE r.vatStatus == "VATABLE"),
    exemptSales: SUM(r.baseRent WHERE r.vatStatus == "EXEMPT"),
    outputVat: SUM(r.vatAmount WHERE r.vatStatus == "VATABLE"),
  }

  form2307Summary = {
    expectedCount: COUNT(r WHERE r.form2307Status != "N/A"),
    receivedCount: COUNT(r WHERE r.form2307Status IN ("RECEIVED", "DELIVERED", "RECONCILED")),
    missingCount: COUNT(r WHERE r.form2307Status == "EXPECTED"),
    totalEwtExpected: SUM(r.ewtWithheld WHERE r.form2307Status != "N/A"),  // from charges
    totalEwtReceived: SUM from form_2307_record received_amount,
  }

  collectionSummary = {
    totalBilled: SUM(r.totalBilled),
    totalCollected: SUM(r.amountCollected),
    totalEwt: SUM(r.ewtWithheld),
    collectionRate: ((totalCollected + totalEwt) / totalBilled × 100).toFixed(1),
  }

  occupancy = {
    totalUnits: rows.length,
    occupiedUnits: occupiedRows.length,
    vacantUnits: rows.length - occupiedRows.length,
    occupancyRate: (occupiedUnits / totalUnits × 100).toFixed(1),
  }

  RETURN { byUnitType, vatSummary, form2307Summary, collectionSummary, occupancy }
```

```
FUNCTION assembleLISRows(propertyId, asOfDate) → LISRow[]:

  activeLeases = query lease

    JOIN rentable ON lease_rentable.rentable_id = rentable.id

      atpNumber: query current active ATP for INVOICE document_type,
      posCrmPermit: null,  // Not applicable for TSVJ
    })
```

### 4.4 SAWT Assembly

```
FUNCTION assembleSAWTRows(quarter) → SAWTRow[]:

  records = query form_2307_record
    JOIN tenant ON form_2307_record.tenant_id = tenant.id
    WHERE form_2307_record.quarter = quarter
      AND form_2307_record.status IN ('RECEIVED', 'DELIVERED', 'RECONCILED')

  seq = 1
  FOR EACH record:
    rows.append(SAWTRow {

      atcCode: "WC157",
      taxBase: record.received_amount / 0.05,  // Gross rent = EWT / rate

      applicableRate: "0.05",
      taxWithheld: record.received_amount,
    })
```

```ts
// Format unit path from room + rentable names
function formatUnitPath(roomName: string, rentableName: string): string {
  // If room is a floor/building and rentable is a unit: "3B", "G1"
  // Concatenation logic based on property structure
  return rentableName; // Simplified — property-specific naming convention
}

// Format lease duration for LIS
function formatDuration(start: string, end: string | null): string {
  if (!end) return "Month-to-month";
  const months = differenceInMonths(new Date(end), new Date(start));
  if (months >= 12 && months % 12 === 0) return `${months / 12} year${months / 12 > 1 ? "s" : ""}`;
  return `${months} months`;
}

// Convert billing period to quarter label
function billingPeriodToQuarter(period: string): string {
  const [year, month] = period.split("-").map(Number);
  const quarter = Math.ceil(month / 3);
  return `${year}-Q${quarter}`;
}

// Compile notes column
function compileNotes(lease: Lease, charges: Charge[]): string {
  const notes: string[] = [];
  if (lease.status === "MONTH_TO_MONTH") notes.push("Month-to-month");
  if (lease.status === "HOLDOVER") notes.push("Holdover");
  if (charges.some(c => c.isProRated)) notes.push("Pro-rated");
  if (charges.some(c => c.creditMemoId)) notes.push("Credit memo applied");
  return notes.join("; ");
}
```

### 4.6 Edge Cases

| Edge Case | Handling |
|-----------|---------|
| **Vacant unit** | Included in rent roll with `tenantName: "(VACANT)"`, all monetary fields `"0.00"`, `notes: "VACANT"`. Excluded from LIS (no tenant). Counted in occupancy denominator. |
| **Multi-unit lease** | One row per lease_rentable combination. Same tenant/TIN appears on multiple rows. Same invoice number if billed on single invoice. Each row shows the rent for that specific rentable. |
| **Mid-month move-in/out** | `totalBilled` reflects actual pro-rated amount from P5 (which already handles pro-ration). `adjustmentFlag = true`. `notes` includes "Pro-rated". |
| **Month-to-month tenant** | `leaseEnd = null`. `leaseStatus = "MONTH_TO_MONTH"`. `leaseDuration = "Month-to-month"` in LIS. Included in rent roll at last applicable rate. |
| **Holdover tenant** | `leaseStatus = "HOLDOVER"`. `notes = "Holdover"`. Rent may be at holdover rate (150-200%) if configured. Included in rent roll but flagged. |
| **2307 partial-quarter** | Corporate tenant starting mid-quarter: `form_2307_record.expected_amount` already reflects actual charges for that quarter (computed by P6). Rent roll shows whatever `form_2307_record.status` is. |
| **Missing TIN** | `tenantTin = null`. Row still generated. AlertBanner on LIS page warns about missing TINs. LIS export proceeds but with blank TIN cells. |
| **Multiple payments in one month** | `amountCollected = SUM` of all payments in period. `receiptNumber` shows first OR number (or null if multiple — UI can show count). |
| **Credit memo in period** | Credit memo creates negative charge in P5. Net effect reflected in `totalBilled`. `adjustmentFlag = true`. `notes` includes "Credit memo applied". |
| **No charges in period** | Row still appears (tenant has active lease) with `totalBilled = "0.00"`. May occur for new leases not yet billed. |
| **Overwritten DRAFT** | Re-generating for same property+period deletes existing DRAFT and creates new. FINALIZED reports cannot be overwritten (CONFLICT error). |
| **Period before any billing data** | Preview/generate works but returns rows with zero charges and payments. Prior balance reflects historical data from `tenant_balance`. |
