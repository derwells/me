# P1 — Rent Escalation Calculation

*Feature Spec | Depends on: F0 (Foundation) | Implementation order: 2 of 15*

---

## Summary

Compute annual rent increases at each lease anniversary, supporting two regimes: NHSB-capped escalation for controlled residential leases (ROUND_DOWN, compounding, max once per 12 months) and contractual escalation for commercial/non-controlled leases (fixed %, stepped, CPI-linked). Track escalation history, manage NHSB cap rate table, detect rent control threshold crossing (PHP 10,000), and create new `RecurringChargePeriod` records that feed into P5 (Monthly Billing Generation).

### 2.4 Procedure Details

#### `escalation.getUpcoming`

1. Query leases WHERE `status IN ('ACTIVE', 'MONTH_TO_MONTH')` AND `escalation_type != 'NONE'`
2. Compute next anniversary date for each lease: `date_start` + N years where N = number of complete years since `date_start`
3. Filter: next anniversary within `daysAhead` (default 90) days of today
4. For NHSB_CAP: look up rate from `nhsb_cap_rate` for the anniversary year. If not found, show "Rate pending" instead of estimate.
5. For contractual: compute rate from `escalation_params`
6. Return sorted by anniversary date ascending

#### `escalation.preview`

1. Load lease with current `monthly_rent`, `escalation_type`, `escalation_params`, `lease_regime`
2. Find the active `recurring_charge` (type=RENT, `is_active=true`) and its latest `recurring_charge_period`
3. Determine the rate:
   - NHSB_CAP: query `nhsb_cap_rate` for the current anniversary year. Fail with `PRECONDITION_FAILED` if not found.
   - FIXED_PCT: use `escalation_params.rate`
   - STEPPED: look up rate for current `yearIndex` (years since lease start)
   - CPI_LINKED: use `escalation_params.baseRate + cpiAdjustment` (CPI adjustment entered manually on params)
4. Call `@tsvj/computations` escalation functions:
   - NHSB_CAP → `computeNHSBEscalation()`
   - Others → `computeContractualEscalation()`
5. Call `checkThresholdCrossing()` on the result
6. Return `EscalationPreview`

#### `escalation.run` (transaction-wrapped)

Within a single `db.transaction()`:

1. **Validate preconditions:**
   - Lease must be ACTIVE or MONTH_TO_MONTH
   - Lease `escalation_type` must not be NONE
   - No escalation event exists for this lease within the last 11 months of `effectiveDate` (enforce max once per 12 months)
   - Active `recurring_charge` with `charge_type_id` = RENT must exist
   - For NHSB_CAP: `nhsb_cap_rate` for the year of `effectiveDate` must exist

2. **Compute escalation** (same as preview, using `@tsvj/computations`)

3. **Close current period:** Set `end_date` on the current `recurring_charge_period` to `effectiveDate - 1 day`

4. **Create new period:**
   ```ts
   INSERT INTO recurring_charge_period (
     recurring_charge_id, period_start, amount, vat_rate
   ) VALUES ($rcId, $effectiveDate, $newRent, $currentVatRate)
   ```

5. **Update lease `monthly_rent`:**
   ```ts
   UPDATE lease SET monthly_rent = $newRent WHERE id = $leaseId
   ```

6. **Handle threshold crossing** (if `thresholdCrossed`):
   - Update `lease.lease_regime` from CONTROLLED_RESIDENTIAL to NON_CONTROLLED_RESIDENTIAL
   - Insert `LeaseEvent` with type `ANNIVERSARY` and `details: { thresholdCrossed: true, previousRegime, newRegime }`
   - Create alert (category: `LEASE_EXPIRY`, level: `WARNING`, title: "Lease regime change", message includes unit and new regime)

7. **Insert EscalationEvent** with all computed values

8. **Insert LeaseEvent** with type `ANNIVERSARY` (even if no threshold crossing — records the escalation)

9. **Audit log** (automatic via middleware)

10. **Invalidate cache:** `escalation.list`, `lease.getById` (rent amount changed)

#### `escalation.runBatch`

For each lease in `leaseIds`, call the same logic as `escalation.run` individually within one transaction. Collect results into `applied` and `skipped` arrays. If any single escalation fails validation, it goes into `skipped` (does not abort the batch).

#### `escalation.upsertNHSBRate`

Simple upsert on `nhsb_cap_rate` by `year`. Uses `ON CONFLICT (year) DO UPDATE`.

#### `escalation.deleteNHSBRate`

Delete by id. Only allowed if no `escalation_event` references this year's rate. Returns `PRECONDITION_FAILED` if referenced.

## 4. Business Logic

### 4.1 Computation Functions (from `@tsvj/computations`)

All escalation computation is delegated to the `@tsvj/computations` package. The tRPC router bridges DB data → computation input → computation output → DB persistence.

**NHSB Escalation** (`computeNHSBEscalation`):
```
newRent = ROUND_DOWN(currentRent × (1 + nhsbCapRate), 2)
```
- Compounding: always uses *current* rent, not original rent
- ROUND_DOWN: truncate toward zero (Crispina `math.py` precedent)
- Threshold: if `newRent > 10000`, `thresholdCrossed = true`

**Contractual Escalation** (`computeContractualEscalation`):
- FIXED_PCT: `newRent = roundMoney(currentRent × (1 + rate))`
- STEPPED: look up rate for year index in schedule, apply like FIXED_PCT
- CPI_LINKED: `newRent = roundMoney(currentRent × (1 + baseRate + cpiAdjustment))`
- NONE: identity (newRent = currentRent)
- Uses `roundMoney` (HALF_UP), not `roundNHSB` (ROUND_DOWN)

**Threshold Check** (`checkThresholdCrossing`):
- Only applies to `unitType = RESIDENTIAL` and `currentRegime = CONTROLLED_RESIDENTIAL`
- If `newMonthlyRent > 10000`, returns `{ crossedThreshold: true, newRegime: "NON_CONTROLLED_RESIDENTIAL" }`

### 4.2 Business Rules Enforced in tRPC Router

| Rule | Enforcement | Source |
|------|-------------|--------|
| Max once per 12 months | Query `escalation_event` for this lease with `event_date` within last 11 months of proposed `effectiveDate`. Reject if found. | RA 9653 Sec. 4 |
| NHSB rate must exist for year | Query `nhsb_cap_rate` WHERE `year = effectiveDate.getFullYear()`. Return `PRECONDITION_FAILED` if not found. | Process catalog P1 |
| Lease must be ACTIVE or MONTH_TO_MONTH | Check `lease.status`. Reject DRAFT, EXPIRED, TERMINATED, RENEWED, HOLDOVER. | Business logic |
| Active recurring charge must exist | Query `recurring_charge` WHERE `lease_id = X AND charge_type_id = RENT AND is_active = true`. | Data model invariant |
| Contractual rate below NHSB cap | For NHSB_CAP only: this is a cap, not a floor. If a lease has a contractual rate *below* the NHSB cap, the NHSB cap still applies (it's the maximum allowed increase). | RA 9653 Sec. 4 |
| Vacancy decontrol | Not handled by escalation — handled at lease creation (new tenant can set any rate). P1 only processes ongoing escalations. | RA 9653 Sec. 4 |

### 4.3 Edge Cases

| Edge Case | Handling |
|-----------|---------|
| **Mid-year NHSB change** | NHSB rate is annual. If NHSB publishes a new rate mid-year, admin updates the rate for the applicable year. Future escalations use the new rate; already-applied escalations are not retroactively changed. |
| **Tacit reconduction (MONTH_TO_MONTH)** | Escalation continues at lease anniversary. The `escalation.run` procedure accepts MONTH_TO_MONTH status. Anniversary date is computed from original `date_start`. |
| **Multiple rentables per lease** | One lease = one monthly rent. The rent applies to all rentables under the lease collectively. Escalation applies to the single `recurring_charge` for the lease, not per-rentable. |
| **Missed anniversary (retroactive)** | Admin can set `effectiveDate` to a past date (the actual anniversary date). The 12-month rule checks relative to `effectiveDate`, not today's date. |
| **NHSB rate not yet published** | `escalation.preview` returns `PRECONDITION_FAILED`. The upcoming table shows "Rate pending" in the estimate column. Admin must add the rate before applying. |
| **Zero-rate escalation** | NONE type or CPI_LINKED with zero adjustment: `newRent = currentRent`. Still creates an EscalationEvent with `escalation_amount = 0` for audit trail. |
| **Threshold crossing mid-batch** | In `runBatch`, each lease is processed independently. A threshold crossing for one lease does not affect others. |

const nhsbRateInput = z.object({

### NHSB Computation (Unit Tests)
- [ ] NHSB escalation uses ROUND_DOWN — `computeNHSBEscalation({ currentRent: "8415.00", nhsbCapRate: "0.0100", ... })` returns `newRent: "8499.00"` (not 8499.15 or 8500.00)

### Contractual Computation (Unit Tests)
- [ ] Fixed percent uses HALF_UP rounding — `computeContractualEscalation({ currentRent: "10000.00", escalationType: "FIXED_PERCENT", escalationParams: { type: "FIXED_PERCENT", rate: "0.05" }, yearIndex: 1 })` returns `newRent: "10500.00"`
