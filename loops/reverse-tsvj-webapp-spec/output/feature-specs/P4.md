# P4 — Late Payment Penalties

*Feature Spec | Depends on: F0 (Foundation) | Implementation order: 7 of 15 (MVP Wave 2: P2+P3+P4)*

---

## Summary

Compute penalty charges on overdue rent and other charges using regime-aware rules: safe harbour caps for controlled residential leases (≤1%/month simple interest, total ≤1 month's rent/year per RA 9653 + Art. 1229) and contractual rates for commercial leases. Track demand letters (required for legal interest when no contractual rate exists). Generate penalty Charge records that feed into P5 (Monthly Billing) and P6 (Payment Tracking). Flag tenants approaching the 3-month arrears ejectment threshold.

### 1.3 Charge Type Seed (from F0)

P4 uses the `PENALTY` charge type seeded in F0:

| Code | Label | is_vatable | default_vat_rate | is_recurring |
|------|-------|:----------:|:----------------:|:------------:|
| PENALTY | Late Payment Penalty | true | 0.12 | false |

VAT on penalty income follows the same rule as the parent rent charge (V5 from cross-cutting extract):
- Commercial lease or residential rent > PHP 15K/month → 12% VAT
- Residential rent ≤ PHP 15K/month → 0% (exempt)

## 2. API Surface

### 2.1 Penalty Router — `apps/web/src/routers/penalty.ts`

All procedures are under the `penalty` sub-router.

#### `penalty.getOverdueCharges`

```ts
input: z.object({
  propertyId: z.number().int().positive().optional(),
  billingPeriod: z.string().regex(/^\d{4}-\d{2}$/),
})
```

Auth: `protectedProcedure` (both roles can view).

Returns all overdue charges with penalty-relevant context for the assessment view:

```ts
output: z.array(z.object({
  leaseId: z.number(),
  tenantId: z.number(),
  tenantName: z.string(),
  leaseRegime: z.enum(["CONTROLLED_RESIDENTIAL", "NON_CONTROLLED_RESIDENTIAL", "COMMERCIAL"]),
  currentMonthlyRent: z.string(),
  penaltyRateMonthly: z.string(),
  gracePeriodDays: z.number(),
  overdueCharges: z.array(z.object({
    chargeId: z.number(),
    chargeType: z.string(),
    billingPeriod: z.string(),
    baseAmount: z.string(),
    unpaidAmount: z.string(),
    dueDate: z.string(),
    daysOverdue: z.number(),
  })),
  totalOverdue: z.string(),
  monthsInArrears: z.number(),
  hasDemandOnFile: z.boolean(),
  latestDemandDate: z.string().nullable(),
  ytdPenaltyTotal: z.string(),
  existingPenaltiesThisPeriod: z.array(z.object({
    penaltyLedgerId: z.number(),
    sourceChargeId: z.number(),
    finalPenalty: z.string(),
  })),
}))
```

**Logic:**
1. Query all charges with status IN ('FINALIZED', 'PARTIALLY_PAID') where `due_date < today` and unpaid balance > 0
2. Group by lease/tenant
3. For each lease, compute: `monthsInArrears` = count of distinct billing periods with unpaid rent charges
4. Look up `demand_record` for each lease (most recent)
5. Sum `penalty_ledger.final_penalty` WHERE `EXTRACT(YEAR FROM created_at) = current year` for YTD total
6. Check existing penalty ledger entries for this billing period (to prevent double-assessment UI)

#### `penalty.preview`

```ts
input: z.object({
  billingPeriod: z.string().regex(/^\d{4}-\d{2}$/),
  leaseId: z.number().int().positive(),
  computationDate: z.string().regex(/^\d{4}-\d{2}-\d{2}$/).optional(), // defaults to today
})
```

Auth: `protectedProcedure`.

Calls `@tsvj/computations` penalty module for each overdue charge under this lease. Returns a preview without persisting anything:

```ts
output: z.object({
  leaseId: z.number(),
  tenantName: z.string(),
  leaseRegime: z.string(),
  currentMonthlyRent: z.string(),
  penaltyRateMonthly: z.string(),
  gracePeriodDays: z.number(),
  hasDemandOnFile: z.boolean(),
  ytdPenaltyTotal: z.string(),
  charges: z.array(z.object({
    chargeId: z.number(),
    chargeType: z.string(),
    billingPeriod: z.string(),
    unpaidAmount: z.string(),
    dueDate: z.string(),
    daysOverdue: z.number(),
    effectiveDaysCharged: z.number(),
    rateApplied: z.string(),
    rawPenalty: z.string(),
    finalPenalty: z.string(),
    penaltyBasis: z.enum(["CONTRACTUAL", "LEGAL_INTEREST", "SAFE_HARBOUR_CAP"]),
    monthlyCapApplied: z.boolean(),
    annualCapApplied: z.boolean(),
  })),
  totalPenalty: z.string(),
  newYtdTotal: z.string(),
  warnings: z.array(z.string()), // e.g., "3-month arrears threshold reached", "No demand on file"
})
```

**Logic:**
1. Fetch lease with penalty_rate_monthly, grace_period_days, lease_regime, monthly_rent
2. Fetch overdue charges (same as getOverdueCharges but scoped to this lease)
3. Fetch latest demand_record for this lease
4. Compute YTD penalty total from penalty_ledger
5. For each overdue charge:
   a. If `lease.penalty_rate_monthly IS NOT NULL` → use contractual rate, penalty runs from due_date + grace_period
   b. If `lease.penalty_rate_monthly IS NULL` AND demand exists → use legal interest (6% p.a. = 0.005/month), runs from demand date
   c. If `lease.penalty_rate_monthly IS NULL` AND no demand → `finalPenalty = "0.00"`, warning: "No contractual rate and no demand on file"
6. Call `computePenalty()` from `@tsvj/computations`
7. Call `applyPenaltyCaps()` for controlled residential leases (running YTD accumulator across charges)
8. Generate warnings: arrears threshold (2 months = warning, 3+ months = ejectment alert for controlled), no demand on file, annual cap approaching

#### `penalty.apply`

```ts
input: z.object({
  billingPeriod: z.string().regex(/^\d{4}-\d{2}$/),
  leaseId: z.number().int().positive(),
  computationDate: z.string().regex(/^\d{4}-\d{2}-\d{2}$/).optional(),
  notes: z.string().max(500).optional(),
})
```

Auth: `adminProcedure`.

Transaction-wrapped. Same computation as `preview`, but persists results:

1. Re-compute penalties (same logic as preview — no stale data)
2. For each charge with `finalPenalty > 0`:
   a. Create `Charge` record with:
      - `charge_type_id` = PENALTY charge type
      - `base_amount` = `finalPenalty`
      - `vat_rate` = determined by VAT rules (same as parent rent charge)
      - `vat_amount` = `roundVAT(finalPenalty × vat_rate)`
      - `total_amount` = `base_amount + vat_amount`
      - `due_date` = last day of `billingPeriod`
      - `status` = FINALIZED
      - `billing_period` = input billingPeriod
      - `description` = `"Late payment penalty on [chargeType] for [sourceBillingPeriod]"`
   b. Create `PenaltyLedger` record with computation details + reference to new Charge
3. If `monthsInArrears >= 3` and `leaseRegime === "CONTROLLED_RESIDENTIAL"`:
   a. Create `Alert` with `category = "ARREARS"`, `level = "OVERDUE"`, entity_type = "lease", entity_id = lease.id
   b. Message: "Tenant [name] has [N] months of unpaid rent — ejectment ground per RA 9653 Sec. 9(b)"
4. Return summary of applied penalties

```ts
output: z.object({
  appliedCount: z.number(),
  totalPenaltyAmount: z.string(),
  totalVatAmount: z.string(),
  chargeIds: z.array(z.number()),
  penaltyLedgerIds: z.array(z.number()),
  arrearsAlertCreated: z.boolean(),
})
```

## 4. Business Logic

### 4.1 Computation Functions (from `@tsvj/computations`)

All penalty computation is delegated to the `@tsvj/computations` penalty module. The tRPC router bridges DB data → computation input → computation output → DB persistence.

**Simple Interest** (`computePenalty`):
```
effectiveDays = max(0, daysOverdue - gracePeriodDays)
rawPenalty = roundPenalty(unpaidAmount × penaltyRateMonthly × (effectiveDays / 30))
```
- Simple interest only — no compounding (Art. 2212: compound interest only from judicial demand)
- Uses `roundPenalty` (HALF_UP, 2 decimals)
- 30-day month standardization (PH lease practice)

**Penalty Caps** (`applyPenaltyCaps`):
- Only for `leaseRegime === "CONTROLLED_RESIDENTIAL"`
- Monthly cap: `min(rawPenalty, roundPenalty(monthlyRent × 0.01))` — max 1% of monthly rent
- Annual cap: `min(afterMonthlyCap, max(0, monthlyRent - totalPenaltiesThisYear))` — total ≤ 1 month's rent per calendar year
- Commercial and non-controlled residential: no caps, rawPenalty returned as-is

### 4.2 Penalty Rate Determination

| Lease Config | Demand Exists? | Applicable Rate | Penalty Basis | Legal Basis |
|-------------|:--------------:|:---------------:|:-------------:|-------------|
| `penalty_rate_monthly` set (≤0.01 controlled) | N/A | Contractual rate | CONTRACTUAL | Art. 1226 + contract |
| `penalty_rate_monthly` set (>0.01 controlled) | N/A | Capped at 0.01 | SAFE_HARBOUR_CAP | Art. 1229 + safe harbour |
| `penalty_rate_monthly` set (any commercial) | N/A | Contractual rate | CONTRACTUAL | Art. 1306 |
| `penalty_rate_monthly` NOT set | Yes | 0.005 (6% p.a. / 12) | LEGAL_INTEREST | Art. 2209 + BSP Circ. 799 |
| `penalty_rate_monthly` NOT set | No | 0 (no penalty) | — | Art. 1956 (must be in writing) |

**Note:** The lease table has `penalty_rate_monthly` defaulting to `0.0100` (1%/month) which covers the most common case. Admin can set it to NULL for leases without a contractual penalty clause, or to a higher rate for commercial leases.

### 4.3 VAT on Penalties

Penalty charges follow the same VAT treatment as the parent rent charge (V5 from cross-cutting extract):
- `unitType = RESIDENTIAL` AND `currentMonthlyRent ≤ PHP 15,000` → `vat_rate = 0.00` (VAT exempt)
- Otherwise → `vat_rate = 0.12` (12% VAT, part of gross receipts per RMC 11-2024 / RR 16-2005)

The VAT rate is determined at penalty Charge creation time via `determineVATRate()` from `@tsvj/computations` with `chargeType = "PENALTY"`.

### 4.4 EWT on Penalties

For corporate tenants (`tenant.is_corporate = true`), penalty income is subject to 5% EWT (same rate as rent, ATC: WC157, per RR 02-98 Sec. 2.57.2(B)). This is handled at payment recording time (P6), not at penalty assessment. The penalty Charge is treated like any other charge for EWT withholding purposes.

### 4.5 Business Rules Enforced in tRPC Router

| Rule | Enforcement | Source |
|------|-------------|--------|
| No double-penalizing same charge in same period | Check `penalty_ledger` for existing entry with same `source_charge_id` + `billing_period`. Skip if found. | Unique constraint + DB check |
| Grace period before penalty accrues | `effectiveDays = max(0, daysOverdue - gracePeriodDays)`. If ≤ 0, no penalty. | RA 9653 Sec. 7 (5 days), contract |
| Controlled safe harbour: ≤1% per month | `applyPenaltyCaps` enforces monthly cap | Art. 1229 + safe harbour advisory |
| Controlled safe harbour: annual total ≤ 1 month's rent | `applyPenaltyCaps` enforces annual cap, accumulating YTD total | Art. 1229 + safe harbour advisory |
| Legal interest only from demand date | If no contractual rate, penalty starts from `demand_record.date_sent`, not charge `due_date` | Art. 2209 + BSP Circ. 799 |
| No penalty without written agreement or demand | If `penalty_rate_monthly IS NULL` AND no demand record → penalty = 0 | Art. 1956 (interest must be in writing) |
| 3-month arrears = ejectment ground (controlled only) | Count distinct billing periods with unpaid rent. Alert at ≥3 months for controlled leases. | RA 9653 Sec. 9(b) |
| Lease must be ACTIVE, MONTH_TO_MONTH, or EXPIRED | Reject penalty assessment on DRAFT, TERMINATED, RENEWED leases | Business logic |

### 4.6 Edge Cases

| Edge Case | Handling |
|-----------|---------|
| **Annual cap already reached** | `applyPenaltyCaps` returns `finalPenalty = "0.00"` with `annualCapApplied = true`. Row shows "Annual cap reached" in UI. Charge is NOT created (no zero-amount charges). |
| **Partial payment on overdue charge** | Penalty computed on unpaid balance (`charge.total_amount - sum(payment_allocation.amount)` for that charge), not original amount. Art. 1229 mandates equitable reduction for partial compliance. |
| **Multiple overdue months** | Each overdue charge gets its own penalty ledger entry with its own `days_overdue` from its own `due_date`. Annual cap accumulates across all penalties in the calendar year. |
| **Demand sent mid-period** | For leases without contractual rate, legal interest runs from `demand_record.date_sent`. If demand sent on 15th and computation on 30th, `daysOverdue = 15` from demand date. |
| **Vacancy decontrol** | When a controlled lease exits rent control (threshold crossing from P1), the penalty regime changes. Next penalty assessment uses the lease's current `lease_regime`. Historical penalties are not retroactively recalculated. |
| **Commercial rate > 3%/month** | Not blocked by the system (freedom of contract), but `penalty.preview` returns a warning: "Rate exceeds 3%/month — may face unconscionability challenge per Art. 1229." Admin can still proceed. |
| **Charge already fully paid after assessment start** | Between `getOverdueCharges` and `apply`, a payment may have been recorded. The `apply` procedure re-checks unpaid balance. If now 0, skip that charge (no stale data penalty). |
| **Leap year / month length** | Standardized 30-day month for penalty computation (common PH lease practice). No per-month adjustment. |
| **Penalty on non-rent charges** | Penalties can be computed on any overdue charge (water, electric, etc.), not just rent. Rate and caps are per-lease, not per-charge-type. |
| **YTD cap across calendar year boundary** | YTD total resets on January 1. A penalty assessed in January for December arrears uses the new year's YTD (starting at 0). |
