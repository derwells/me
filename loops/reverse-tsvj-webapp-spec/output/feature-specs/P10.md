# P10 — Lease Status Visibility

*Feature Spec | Depends on: F0, P1, P5, P6, P9 | Implementation order: 11 of 15 (batch P8+P9+P10)*

---

## Summary

Portfolio-level lease monitoring with automated lifecycle processing. Provides a real-time portfolio dashboard showing lease counts by status, expiring leases, reconduction watch, revenue and occupancy metrics, and PFRS 16 maturity analysis export. Implements the lease lifecycle section of the daily `process_daily_alerts()` PL/pgSQL function — auto-transitioning ACTIVE→EXPIRED and EXPIRED→MONTH_TO_MONTH (tacit reconduction per Art. 1670), generating expiry alerts at 90/60/30/15 days, and detecting lease anniversaries. P10 is read-heavy with no mutations — all 8 new procedures are queries.

### 1.3 SQL Migration — Lease Lifecycle in `process_daily_alerts()`

P10 adds the lease lifecycle section to the existing `process_daily_alerts()` PL/pgSQL function (shell created by F0, extended by P5/P6 for ATP and arrears). This section runs daily at 00:05 PHT via pg_cron.

**File:** `supabase/migrations/YYYYMMDDHHMMSS_add_lease_lifecycle_to_daily_alerts.sql`

```sql
CREATE OR REPLACE FUNCTION process_daily_alerts() RETURNS void AS $$
DECLARE
  rec RECORD;
  v_alert_level TEXT;
BEGIN
  -- ... (existing sections from F0, P5, P6 preserved above) ...

  -- ============================================
  -- SECTION: Lease Lifecycle (P10)
  -- ============================================

  -- 1. ACTIVE → EXPIRED (date_end passed)
  FOR rec IN
    SELECT id, tenant_id FROM lease
    WHERE status = 'ACTIVE' AND date_end < CURRENT_DATE
  LOOP
    UPDATE lease SET status = 'EXPIRED', updated_at = NOW() WHERE id = rec.id;

    INSERT INTO lease_event (lease_id, event_type, event_date, triggered_by, created_at)
      VALUES (rec.id, 'EXPIRED', CURRENT_DATE, 'SYSTEM', NOW());

    INSERT INTO alert (level, category, title, message, entity_type, entity_id, created_at)
      SELECT 'WARNING', 'LEASE_EXPIRY',
        'Lease expired',
        'Lease #' || rec.id || ' has expired. 15-day reconduction window begins.',
        'lease', rec.id, NOW()
      WHERE NOT EXISTS (
        SELECT 1 FROM alert
        WHERE category = 'LEASE_EXPIRY' AND entity_type = 'lease' AND entity_id = rec.id
          AND title = 'Lease expired'
          AND created_at > NOW() - INTERVAL '1 day'
      );
  END LOOP;

  -- 2. EXPIRED → MONTH_TO_MONTH (tacit reconduction, Art. 1670)
  -- Conditions: 15+ days since expiry, no non-renewal notice, not already reconduceted
  FOR rec IN
    SELECT l.id, l.tenant_id, l.has_guarantor, l.lease_regime
    FROM lease l
    WHERE l.status = 'EXPIRED'
      AND (CURRENT_DATE - l.date_end) >= 15
      AND NOT EXISTS (
        SELECT 1 FROM non_renewal_notice n WHERE n.lease_id = l.id
      )
      AND NOT EXISTS (
        SELECT 1 FROM lease_event e
        WHERE e.lease_id = l.id AND e.event_type = 'RECONDUCTION_STARTED'
      )
  LOOP
    UPDATE lease SET status = 'MONTH_TO_MONTH', updated_at = NOW() WHERE id = rec.id;

    INSERT INTO lease_event (lease_id, event_type, event_date, triggered_by, details, created_at)
      VALUES (rec.id, 'RECONDUCTION_STARTED', CURRENT_DATE, 'SYSTEM',
        jsonb_build_object(
          'guarantor_released', rec.has_guarantor,
          'regime', rec.lease_regime
        ),
        NOW());

    INSERT INTO alert (level, category, title, message, entity_type, entity_id, created_at)
      VALUES ('WARNING', 'LEASE_EXPIRY',
        'Tacit reconduction',
        'Lease #' || rec.id || ' has entered month-to-month (Art. 1670).'
          || CASE WHEN rec.has_guarantor THEN ' Guarantor released (Art. 1672).' ELSE '' END,
        'lease', rec.id, NOW());
  END LOOP;

  -- 3. Expiry alerts at 90/60/30/15 days before date_end
  FOR rec IN
    SELECT id, date_end, (date_end - CURRENT_DATE) AS days_to_expiry
    FROM lease
    WHERE status = 'ACTIVE'
      AND (date_end - CURRENT_DATE) IN (90, 60, 30, 15)
  LOOP
    v_alert_level := CASE
      WHEN rec.days_to_expiry = 90 THEN 'INFO'
      WHEN rec.days_to_expiry = 60 THEN 'WARNING'
      WHEN rec.days_to_expiry = 30 THEN 'URGENT'
      WHEN rec.days_to_expiry = 15 THEN 'OVERDUE'
    END;

    -- Deduplicate: no same alert for same entity within 24 hours
    INSERT INTO alert (level, category, title, message, entity_type, entity_id, created_at)
      SELECT v_alert_level, 'LEASE_EXPIRY',
        'Lease expiring in ' || rec.days_to_expiry || ' days',
        'Lease #' || rec.id || ' expires on ' || rec.date_end
          || '. Action required: renew or send non-renewal notice.',
        'lease', rec.id, NOW()
      WHERE NOT EXISTS (
        SELECT 1 FROM alert
        WHERE category = 'LEASE_EXPIRY' AND entity_type = 'lease' AND entity_id = rec.id
          AND title = 'Lease expiring in ' || rec.days_to_expiry || ' days'
          AND created_at > NOW() - INTERVAL '1 day'
      );
  END LOOP;

  -- 4. Anniversary detection (lease start date anniversary)
  FOR rec IN
    SELECT id, date_start
    FROM lease
    WHERE status IN ('ACTIVE', 'MONTH_TO_MONTH')
      AND EXTRACT(MONTH FROM CURRENT_DATE) = EXTRACT(MONTH FROM date_start)
      AND EXTRACT(DAY FROM CURRENT_DATE) = EXTRACT(DAY FROM date_start)
      AND date_start < CURRENT_DATE  -- not the start day itself
  LOOP
    -- Deduplicate: no duplicate anniversary event for same date
    IF NOT EXISTS (
      SELECT 1 FROM lease_event
      WHERE lease_id = rec.id AND event_type = 'ANNIVERSARY'
        AND event_date = CURRENT_DATE
    ) THEN
      INSERT INTO lease_event (lease_id, event_type, event_date, triggered_by, created_at)
        VALUES (rec.id, 'ANNIVERSARY', CURRENT_DATE, 'SYSTEM', NOW());

      INSERT INTO alert (level, category, title, message, entity_type, entity_id, created_at)
        SELECT 'INFO', 'LEASE_EXPIRY',
          'Lease anniversary',
          'Lease #' || rec.id || ' anniversary — rent escalation check due.',
          'lease', rec.id, NOW()
        WHERE NOT EXISTS (
          SELECT 1 FROM alert
          WHERE category = 'LEASE_EXPIRY' AND entity_type = 'lease' AND entity_id = rec.id
            AND title = 'Lease anniversary'
            AND created_at > NOW() - INTERVAL '1 day'
        );
    END IF;
  END LOOP;

END;
$$ LANGUAGE plpgsql;
```

#### `portfolio.getSummary`

Returns aggregate portfolio metrics for dashboard KPI cards.

**Input:**
```ts
z.object({
  propertyId: z.number().int().positive().optional(),
})
```

**Output:**
```ts
z.object({
  statusCounts: z.object({
    active: z.number(),
    expiring: z.number(),        // ACTIVE leases with date_end within 90 days
    monthToMonth: z.number(),
    holdover: z.number(),
    terminated: z.number(),
    draft: z.number(),
    renewed: z.number(),
  }),
  regimeCounts: z.object({
    controlled: z.number(),
    nonControlled: z.number(),
    commercial: z.number(),
  }),
  revenue: z.object({
    active: z.string(),          // sum of current rent for ACTIVE leases (Peso)
    monthToMonth: z.string(),    // sum for MONTH_TO_MONTH leases
    total: z.string(),           // active + monthToMonth
  }),
  occupancy: z.object({
    totalUnits: z.number(),
    occupiedUnits: z.number(),
    vacantUnits: z.number(),
    occupancyRate: z.string(),   // percentage, e.g. "85.7"
  }),
  weightedAvgRemainingMonths: z.string(),
  leasesRequiringAttention: z.number(),  // undismissed LEASE_EXPIRY alert count
})
```

**Implementation:**
1. Status counts: `SELECT status, COUNT(*) FROM lease GROUP BY status` (exclude nothing — show all statuses)
2. Expiring: `SELECT COUNT(*) FROM lease WHERE status = 'ACTIVE' AND date_end <= CURRENT_DATE + 90`
3. Regime counts: `SELECT lease_regime, COUNT(*) FROM lease WHERE status IN ('ACTIVE', 'MONTH_TO_MONTH') GROUP BY lease_regime`
4. Revenue: Join `lease` → `recurring_charge` (is_active=true, charge_type=RENT) → `recurring_charge_period` (is latest by period_start). Sum `amount` grouped by `lease.status`.
5. Occupancy: Total = `COUNT(*) FROM rentable WHERE is_active = true`. Occupied = `COUNT(DISTINCT lr.rentable_id) FROM lease_rentable lr JOIN lease l ON l.id = lr.lease_id WHERE l.status IN ('ACTIVE', 'MONTH_TO_MONTH')`.
6. Weighted avg remaining term: Fetch active/M-T-M leases with rent + remaining months, pass to `computeWeightedAvgRemainingTerm` from `@tsvj/computations`.
7. Requiring attention: `SELECT COUNT(*) FROM alert WHERE is_dismissed = false AND category = 'LEASE_EXPIRY'`

```ts
    isReconductionWindow: z.boolean(), // EXPIRED && daysSinceExpiry < 15
    hasGuarantor: z.boolean(),
  })),
  nextCursor: z.number().nullable(),

```

**Implementation:** Multi-table JOIN:
```sql
SELECT l.id AS lease_id, l.status, l.lease_regime, l.date_start, l.date_end,
  l.escalation_type, l.has_guarantor,
  (l.date_end - CURRENT_DATE) AS days_to_expiry,
  t.id AS tenant_id, t.full_name AS tenant_name, t.tin AS tenant_tin, t.is_corporate,
  p.name AS property_name,
  CONCAT_WS(' > ', p.name, rm.name, r.name) AS unit_designation,
  rcp.amount AS monthly_rent,
  COALESCE(tb.total_billed - tb.total_paid - tb.total_ewt, 0) AS total_balance,
  COALESCE(tb.months_in_arrears, 0) AS months_in_arrears,
  (nrn.id IS NOT NULL) AS has_non_renewal_notice,
  rr.id AS renewal_record_id,
  (l.status = 'EXPIRED' AND (CURRENT_DATE - l.date_end) < 15) AS is_reconduction_window,
  -- next_escalation_date: date_start + N years where N = years since start + 1
  (l.date_start + (EXTRACT(YEAR FROM age(CURRENT_DATE, l.date_start))::int + 1)
    * INTERVAL '1 year')::date AS next_escalation_date
FROM lease l
  JOIN tenant t ON t.id = l.tenant_id
  LEFT JOIN lease_rentable lr ON lr.lease_id = l.id
  LEFT JOIN rentable r ON r.id = lr.rentable_id
  LEFT JOIN room rm ON rm.id = r.room_id
  LEFT JOIN property p ON p.id = rm.property_id
  LEFT JOIN LATERAL (
    SELECT rcp2.amount FROM recurring_charge rc
    JOIN recurring_charge_period rcp2 ON rcp2.recurring_charge_id = rc.id
    WHERE rc.lease_id = l.id AND rc.is_active = true
    ORDER BY rcp2.period_start DESC LIMIT 1
  ) rcp ON true
  LEFT JOIN tenant_balance tb ON tb.tenant_id = t.id
  LEFT JOIN non_renewal_notice nrn ON nrn.lease_id = l.id
  LEFT JOIN renewal_record rr ON rr.predecessor_lease_id = l.id
WHERE 1=1
  AND ($status IS NULL OR l.status = $status)
  AND ($leaseRegime IS NULL OR l.lease_regime = $leaseRegime)
  AND ($propertyId IS NULL OR p.id = $propertyId)
  AND ($tenantId IS NULL OR t.id = $tenantId)
  AND ($expiringWithinDays IS NULL OR (l.status = 'ACTIVE' AND l.date_end <= CURRENT_DATE + $expiringWithinDays))
  AND ($hasBalance IS NULL OR ($hasBalance = true AND COALESCE(tb.total_billed - tb.total_paid - tb.total_ewt, 0) > 0))
  AND ($cursor IS NULL OR l.id > $cursor)
ORDER BY ... -- dynamic based on sortBy
LIMIT $limit + 1
```

For leases with multiple rentables: use the first rentable by `lease_rentable.id` order. The unit designation shows one unit; if multiple exist, append ` (+N)`.

#### `portfolio.getReconductionWatch`

EXPIRED leases currently in the 15-day reconduction window (Art. 1670). Critical decision point: admin must renew, send non-renewal notice, or allow tacit reconduction.

**Input:** none

**Output:**
```ts
z.object({
  items: z.array(z.object({
    leaseId: z.number(),
    tenantName: z.string(),
    unitDesignation: z.string(),
    dateEnd: z.string(),
    daysSinceExpiry: z.number(),
    daysRemaining: z.number(),     // 15 - daysSinceExpiry (urgency indicator)
    hasNotice: z.boolean(),
    noticeDate: z.string().nullable(),
    leaseRegime: leaseRegimeEnum,
    guarantorAtRisk: z.boolean(),  // has_guarantor AND controlled → Art. 1672 release
    monthlyRent: z.string(),
  })),
})
```

**Implementation:**
```sql
SELECT l.id, t.full_name, CONCAT_WS(' > ', p.name, rm.name, r.name),
  l.date_end, l.lease_regime, l.has_guarantor,
  (CURRENT_DATE - l.date_end) AS days_since_expiry,
  (15 - (CURRENT_DATE - l.date_end)) AS days_remaining,
  (nrn.id IS NOT NULL) AS has_notice,
  nrn.sent_date AS notice_date,
  (l.has_guarantor AND l.lease_regime = 'CONTROLLED_RESIDENTIAL') AS guarantor_at_risk,
  rcp.amount AS monthly_rent
FROM lease l
  JOIN tenant t ON t.id = l.tenant_id
  LEFT JOIN lease_rentable lr ON lr.lease_id = l.id
  LEFT JOIN rentable r ON r.id = lr.rentable_id
  LEFT JOIN room rm ON rm.id = r.room_id
  LEFT JOIN property p ON p.id = rm.property_id
  LEFT JOIN non_renewal_notice nrn ON nrn.lease_id = l.id
  LEFT JOIN LATERAL (...) rcp ON true
WHERE l.status = 'EXPIRED'
  AND (CURRENT_DATE - l.date_end) BETWEEN 0 AND 14
ORDER BY (CURRENT_DATE - l.date_end) DESC  -- most urgent first
```

**Implementation:**
```sql
-- Total units per property
SELECT p.id, p.name, COUNT(r.id) AS total_units
FROM property p
  JOIN room rm ON rm.property_id = p.id
  JOIN rentable r ON r.room_id = rm.id
WHERE r.is_active = true
GROUP BY p.id, p.name

-- Occupied units: rentable has active lease
SELECT p.id, COUNT(DISTINCT r.id) AS occupied_units
FROM property p
  JOIN room rm ON rm.property_id = p.id
  JOIN rentable r ON r.room_id = rm.id
  JOIN lease_rentable lr ON lr.rentable_id = r.id
  JOIN lease l ON l.id = lr.lease_id
WHERE l.status IN ('ACTIVE', 'MONTH_TO_MONTH')
  AND r.is_active = true
GROUP BY p.id
```

Vacant = total - occupied. Rate = `(occupied / total * 100)` rounded HALF_UP to 1 decimal.

## 4. Business Logic

### 4.1 Computation Functions — `@tsvj/computations` lease module

P10 adds two functions to the existing `@tsvj/computations` lease module (which already contains `isValidTransition` and `computeExpiryAlerts` from F0).

**`computeWeightedAvgRemainingTerm`:**

Calculates rent-weighted average remaining lease term in months.

```ts
export function computeWeightedAvgRemainingTerm(
  leases: Array<{ monthlyRent: string; remainingMonths: number }>,
): string {
  if (leases.length === 0) return "0.0";

  let totalWeight = new Peso("0");
  let weightedSum = new Peso("0");

  for (const lease of leases) {
    const rent = new Peso(lease.monthlyRent);
    totalWeight = totalWeight.add(rent);
    weightedSum = weightedSum.add(rent.multiply(lease.remainingMonths));
  }

  if (totalWeight.isZero()) return "0.0";
  return weightedSum.dividedBy(totalWeight).toFixed(1, Decimal.ROUND_HALF_UP);
}
```

- Input: Array of `{ monthlyRent, remainingMonths }` for ACTIVE and MONTH_TO_MONTH leases
- Month-to-month leases: `remainingMonths = 1` (cancellable)
- Active leases: `remainingMonths = max(0, monthsBetween(asOfDate, dateEnd))`
- Output: String with 1 decimal place, HALF_UP rounding

**`computePFRS16Maturity`:**

Computes undiscounted future lease payments by year band (1-5 + thereafter) per PFRS 16 para 97.

```ts
export function computePFRS16Maturity(
  lease: {
    monthlyRent: string;
    dateEnd: string;
    status: "ACTIVE" | "MONTH_TO_MONTH";
  },
  asOfDate: string,
): {
  year1: string;
  year2: string;
  year3: string;
  year4: string;
  year5: string;
  thereafter: string;
  total: string;
} {
  const rent = new Peso(lease.monthlyRent);

  let remainingMonths: number;
  if (lease.status === "MONTH_TO_MONTH") {
    remainingMonths = 1;
  } else {
    remainingMonths = Math.max(
      0,
      differenceInMonths(new Date(lease.dateEnd), new Date(asOfDate)),
    );
  }

  const bands = [0, 0, 0, 0, 0]; // years 1-5
  let thereafter = 0;

  for (let m = 1; m <= remainingMonths; m++) {
    const yearIndex = Math.ceil(m / 12) - 1; // 0-4 for years 1-5
    if (yearIndex < 5) {
      bands[yearIndex]++;
    } else {
      thereafter++;
    }
  }

  return {
    year1: rent.multiply(bands[0]).toRoundedString(),
    year2: rent.multiply(bands[1]).toRoundedString(),
    year3: rent.multiply(bands[2]).toRoundedString(),
    year4: rent.multiply(bands[3]).toRoundedString(),
    year5: rent.multiply(bands[4]).toRoundedString(),
    thereafter: rent.multiply(thereafter).toRoundedString(),
    total: rent.multiply(remainingMonths).toRoundedString(),
  };
}
```

- Uses current monthly rent (does not project future escalations — MVP simplification)
- Month-to-month: 1 month in Year 1 band only
- All monetary values use `roundMoney` (HALF_UP to 2 decimal places)

### 4.2 Daily Job Business Rules

| Rule | Enforcement | Source |
|------|-------------|--------|
| ACTIVE→EXPIRED only when date_end < today | PL/pgSQL `WHERE date_end < CURRENT_DATE` | Business logic |
| EXPIRED→MONTH_TO_MONTH requires 15 days | PL/pgSQL `WHERE (CURRENT_DATE - date_end) >= 15` | Art. 1670 Civil Code |
| Non-renewal notice blocks reconduction | PL/pgSQL `NOT EXISTS (SELECT 1 FROM non_renewal_notice)` | Art. 1670 (notice = no acquiescence) |
| No duplicate reconduction | PL/pgSQL `NOT EXISTS (SELECT 1 FROM lease_event WHERE event_type = 'RECONDUCTION_STARTED')` | Idempotency |
| Expiry alerts at exactly 90/60/30/15 | PL/pgSQL `WHERE (date_end - CURRENT_DATE) IN (90, 60, 30, 15)` | Process catalog P10 |
| Alert deduplication | `NOT EXISTS ... created_at > NOW() - INTERVAL '1 day'` | compliance-alerts analysis |
| Anniversary = same month + day as date_start | PL/pgSQL EXTRACT comparison | P1 escalation trigger |
| Guarantor release logged at reconduction | JSONB `{ guarantor_released: true }` in lease_event details | Art. 1672 Civil Code |

### 4.3 Edge Cases

| Edge Case | Handling |
|-----------|---------|
| **Lease with no rentable** | Occupancy query uses `LEFT JOIN lease_rentable`. A lease without a rentable is included in status counts but not in occupancy calculation. |
| **Multiple rentables per lease** | Portfolio list shows primary rentable (first by `lease_rentable.id`) with `(+N)` suffix. Occupancy counts each rentable independently. |
| **Month-to-month in PFRS 16** | Treated as 1-month remaining term. Falls entirely in Year 1 band. Identified separately in output with `status: 'MONTH_TO_MONTH'`. |
| **February 29 anniversary** | PL/pgSQL EXTRACT(MONTH) + EXTRACT(DAY) comparison. On non-leap years, Feb 29 leases have no anniversary. Admin can trigger P1 escalation manually via escalation.run for the nearest date. |
| **Vacancy decontrol at reconduction** | When EXPIRED→MONTH_TO_MONTH, the regime does NOT change. Vacancy decontrol only applies at lease creation (new tenant). Art. 1670 says terms continue as-is. |
| **Reconduction window boundary** | Day 0 = expiry date, Day 14 = last day in window, Day 15 = auto-reconduction fires. `BETWEEN 0 AND 14` for watch panel, `>= 15` for auto-transition. |
| **Holdover lease** | HOLDOVER is a manual status (commercial only). Daily job does not auto-create HOLDOVER. Admin sets via lease.update. Portfolio dashboard shows holdover count. |
| **Already reconduceted lease** | Daily job checks `NOT EXISTS lease_event WHERE event_type = 'RECONDUCTION_STARTED'` to prevent double processing. Idempotent. |
| **Concurrent materialized view refresh** | P10 queries `tenant_balance` but does not refresh it. Refresh happens in P6 (payment) and P5 (billing) mutations. Stale data acceptable for dashboard (30s staleTime). |
| **Large portfolio** | All queries use indexes (status, date_end, tenant_id). Cursor-based pagination for full list. KPI cards are aggregate queries (fast). No full table scans. |
