# F0 — Foundation: Lease & Tenant Master

*Feature Spec | Depends on: None (first feature) | Implementation order: 1 of 15*

---

## Summary

Set up the Turborepo monorepo, database schema for all foundation entities (Property, Room, Rentable, Tenant, Lease, ChargeType), authentication infrastructure (Supabase Auth, login, role enforcement), tRPC API layer, app shell (sidebar, header, alert bell), CRUD views for tenants/leases/properties, dashboard skeleton, settings pages, seed data, and CI/CD pipeline. This is the base layer that every subsequent feature builds on.

---

export const propertyRelations = relations(property, ({ many }) => ({
  rooms: many(room),

  property: one(property, { fields: [room.propertyId], references: [property.id] }),
  rentables: many(rentable),
}));

export const rentableRelations = relations(rentable, ({ one, many }) => ({
  room: one(room, { fields: [rentable.roomId], references: [room.id] }),
  property: one(property, { fields: [rentable.propertyId], references: [property.id] }),
  leaseRentables: many(leaseRentable),
}));

export const tenantRelations = relations(tenant, ({ many }) => ({
  leases: many(lease),
}));

export const leaseRelations = relations(lease, ({ one, many }) => ({
  tenant: one(tenant, { fields: [lease.tenantId], references: [tenant.id] }),
  parentLease: one(lease, { fields: [lease.parentLeaseId], references: [lease.id] }),
  leaseRentables: many(leaseRentable),
  events: many(leaseEvent),
}));

export const leaseRentableRelations = relations(leaseRentable, ({ one }) => ({
  lease: one(lease, { fields: [leaseRentable.leaseId], references: [lease.id] }),
  rentable: one(rentable, { fields: [leaseRentable.rentableId], references: [rentable.id] }),

export const leaseEventRelations = relations(leaseEvent, ({ one }) => ({
  lease: one(lease, { fields: [leaseEvent.leaseId], references: [lease.id] }),

**leaseCreateInput:**
```ts
z.object({
  tenantId: z.number(),
  rentableIds: z.array(z.number()).min(1),
  leaseRegime: z.enum(["CONTROLLED_RESIDENTIAL", "NON_CONTROLLED_RESIDENTIAL", "COMMERCIAL"]),
  dateStart: z.string(),  // ISO date
  dateEnd: z.string(),
  monthlyRent: z.string().regex(/^\d+(\.\d{1,2})?$/),  // Decimal string
  customDueDay: z.number().min(1).max(31).default(5),
  escalationType: z.enum(["NHSB_CAP", "FIXED_PCT", "STEPPED", "CPI_LINKED", "NONE"]),
  escalationParams: escalationParamsSchema.optional(),
  penaltyRateMonthly: z.string().default("0.0100"),
  gracePeriodDays: z.number().default(5),
  depositMonths: z.number().default(2),
  advanceMonths: z.number().default(1),
  notes: z.string().optional().nullable(),
}).superRefine((data, ctx) => {
  // Controlled residential constraints
  if (data.leaseRegime === "CONTROLLED_RESIDENTIAL") {
    if (data.depositMonths > 2) {
      ctx.addIssue({
        code: z.ZodIssueCode.custom,
        message: "Controlled residential: max 2 months deposit (RA 9653 Sec. 7)",
        path: ["depositMonths"],
      });
    }
    if (data.advanceMonths > 1) {
      ctx.addIssue({
        code: z.ZodIssueCode.custom,
        message: "Controlled residential: max 1 month advance (RA 9653 Sec. 7)",
        path: ["advanceMonths"],
      });
    }
  }
  // Date validation
  if (new Date(data.dateEnd) <= new Date(data.dateStart)) {
    ctx.addIssue({
      code: z.ZodIssueCode.custom,
      message: "End date must be after start date",
      path: ["dateEnd"],
    });
  }
})
```

**Business rules enforced in mutation:**
- `lease.create`: Determine `vat_rate` from `leaseRegime` + `monthlyRent` (see cross-cutting §1.1). Insert lease + lease_rentable records + LeaseEvent(CREATED) in one transaction.
- `lease.activate`: Transition DRAFT→ACTIVE. Insert LeaseEvent(ACTIVATED). Validate lease has at least one rentable assigned.
- `lease.terminate`: Transition ACTIVE/MONTH_TO_MONTH/HOLDOVER→TERMINATED. Insert LeaseEvent(TERMINATED) with reason in details. Only admin.

## 5. Business Logic

### 5.1 VAT Determination at Lease Creation

When a lease is created, `vat_rate` is determined:
```
IF lease_regime = CONTROLLED_RESIDENTIAL AND monthly_rent <= 15000:
    vat_rate = 0.00  (exempt — NIRC 109(1)(Q))
ELIF lessor_aggregate_annual_receipts <= 3000000:
    vat_rate = 0.00  (below threshold)
ELSE:
    vat_rate = 0.12  (12% VAT)
```

This is a display-only computation in F0. The `vat_rate` is frozen on the lease for reference. Actual per-charge VAT is determined at billing time (P5) using the full 8-scenario matrix from cross-cutting §1.

### 5.2 Lease Regime Determination

The lease creation form auto-suggests regime based on `rentable.unit_type`:
- If ALL selected rentables are RESIDENTIAL → suggest CONTROLLED_RESIDENTIAL (if rent ≤ PHP 10K) or NON_CONTROLLED_RESIDENTIAL
- If ANY selected rentable is COMMERCIAL → suggest COMMERCIAL
- Admin can override (e.g., vacancy decontrol pricing above PHP 10K for a residential unit)

### 5.3 Lease State Machine (F0 Subset)

F0 implements manual transitions only. Automatic transitions (daily cron) are in P10.

| From | To | F0 Action |
|------|----|-----------|
| — | DRAFT | `lease.create` |
| DRAFT | ACTIVE | `lease.activate` |
| ACTIVE/MONTH_TO_MONTH/HOLDOVER | TERMINATED | `lease.terminate` |

Each transition inserts a `LeaseEvent` with the appropriate event type and triggered_by = current user ID.

### 5.4 Audit Logging

The `withAuditLog` middleware auto-logs all admin mutations:
- `action`: tRPC procedure path (e.g., "tenant.create")
- `entityType`: first segment of path (e.g., "tenant")
- `userId`: from auth context
- Fire-and-forget (non-blocking)

Individual procedures can enrich with `entityId` and `details` by writing to `auditLog` directly within the transaction.
