## Recently Analyzed
- [x] classification-code-usage (Wave 2) — 2026-03-03 — 76,577 classified data rows extracted from 31 workbooks. 62 unique codes found: all 13 primary Annex B codes (100%), 43 of 50 agricultural sub-codes (86%), and 6 non-standard regional codes (WC, AR, PC, PH, R, A0). Stark NCR/Provincial divide: NCR is a 7-code system ({RR, CR, RC, CC, PS, X, I} = 98.5% of 47K rows, zero agricultural sub-codes in pure NCR RDOs), while provincial is a 59-code system (agricultural codes = 42.3% of 29.5K rows). Top 3 codes (RR 38.2%, CR 24.5%, RC 5.0%) = 67.7% of all rows. RDO 113A (Davao) is most diverse at 43 codes; RDO 30 (Binondo) is minimal at 5 codes (no RR — purely commercial). 2,739+ rows have footnote-embedded classifications (CODE*, *CODE, CODE***) requiring asterisk stripping. Cebu RDO 83 embeds revision numbers in classification cells ("A50 23*"). Classification legends in workbook preambles (rows 79-120) vary per revision — critical for legacy code mapping. CR is the only universal code (31/31 RDOs). Parser design: 64-variant Rust enum, normalization pipeline (strip asterisks → strip revision numbers → split slashes → collapse spaces → lookup standard map), per-RDO available_codes_index for UI filtering.
- [x] address-vicinity-patterns (Wave 2) — 2026-03-03 — 28,109 vicinity records extracted from 31 workbooks (13,080 unique values). Two fundamentally different address models confirmed: NCR cross-street boundary segments (40.7% of records, separator types: spaced hyphen 3,725, tight hyphen 2,443, "TO" 419) vs. provincial road-proximity hierarchy (7 tiers: National Highway → Provincial Road → Municipal Road → Barangay Road → 50m Inward → Interior → Watershed, with ZV ranges ₱100–₱35,000). 14 pattern categories identified and consolidated to 10 semantic types. Key discoveries: (1) 331 cross-streets with 3+ segments create separator ambiguity requiring street name dictionary, (2) 211 former-name annotations enable alias resolution (VITO CRUZ = PABLO OCAMPO SR.), (3) RDO 43 Pasig uses semicolon-delimited barangay qualifiers in vicinity column (248 entries), (4) RDO 44 BGC uses FAR 1-18 tiers instead of streets (544 entries), (5) 1,209 asterisk-prefixed cross-streets must strip footnote markers before matching, (6) RDO 46 Cainta-Taytay is a hybrid zone mixing both models. Provincial road-proximity tiers extracted with ZV values: T1 National median ₱12K, T4 Barangay median ₱4K, T6 Interior median ₱3.5K, T7 Watershed median ₱500. Parser pseudocode produced for vicinity type detection, cross-street parsing, road-tier resolution, and normalization pipeline.
- [x] sheet-organization (Wave 2) — 2026-03-03 — 294 total sheets across 31 workbooks: 31 NOTICE + 261 revision data + 2 empty artifacts. Universal pattern: NOTICE at index 0, then Sheet N (DO NNN-YY) in descending order (newest first). No condo/land separation — all data types mixed in every sheet. No hidden sheets. 25 naming anomalies (unclosed parens, missing DO prefix, whitespace). 30 Department Orders shared across RDO clusters (Makati 4-way, South NCR 3-way, QC 3-way) revealing historical RDO splits. Provincial workbooks use NOTICE "City/Municipalities Covered" column to map per-municipality revisions — different municipalities can be on different current revisions within the same workbook. RDO-56 (Laguna) has maximum 17 revision sheets. RDO-57 has duplicate DO number (DO 11-15 in two sheets, different municipalities). 3 effectivity date formats (Excel serial 86%, ISO 9%, human 5%). Column counts vary across revisions in every workbook — parser must detect layout per-sheet. Growth analysis: Pangasinan RDOs show 25.7x growth (795→14,609 rows over 33 years). Parser design: NOTICE table is authoritative source for DO→sheet→municipality→effectivity mapping; unique key is (RDO, DO number), not DO alone.
- [x] merged-cell-patterns (Wave 2) — 2026-03-03 — 52,327 total merges across 31 workbooks (31,237 in current revision sheets). 6 merge categories: VERT 72%, FULL_H 11.2%, TRIPLE_H 9.0%, PAIR_H 4.9%, BLOCK_H 2.3%, BLOCK 0.8%. Critical distinction: header repetition merges (column headers spanning 2-3 rows at barangay boundaries — safe to skip) vs. data row merges (street/vicinity spanning multiple classification rows — must resolve). 12 of 31 workbooks (39%) use data vertical merges; RDO 32 has 2,533 data merges, RDO 34 has 2,187. Pattern E (RDO 33, 81) uses horizontal merges for every data row (street spans cols 0-2, vicinity spans cols 3-4). Pattern D (RDO 30) uses multi-row block merges for condo names. Extreme cases: 17-row merge (RDO 53B, EAST SERVICE ROAD), 14-row agricultural merge (RDO 4). MergeMap data structure designed for O(1) cell resolution. Parser must construct merge map before reading data.
- [x] workbook-column-layouts (Wave 2) — 2026-03-03 — 31 workbooks parsed for exact column positions. 6 distinct column layout patterns identified: Pattern A (standard 4-col at 0,1,2,3) covers 81% (25/31).
