## Executive Summary

The proposed 6-level fallback hierarchy is **STRUCTURALLY SOUND but requires renumbering and one significant correction.** The hierarchy as presented conflates two different source documents' ordering. Specifically:

1. **Levels 1-3 are CONFIRMED** by 3+ independent sources each.
2. **Level 4 (Adjacent Barangay) is CONFIRMED** but its position relative to the institutional fallback (Level 5) needs clarification — they address different problem domains (geographic gap vs. classification gap) and can fire in parallel rather than sequentially.
3. **Level 5 (Institutional → Commercial) is CONFIRMED** with a critical authority attribution correction: the rule comes from **DOF Department Order footnotes**, NOT from RMO 31-2019 or RAMO 2-91.
4. **Level 6 (NULL / No Match) is CONFIRMED** by CTA case law.
5. **The RAMO 2-91 LGU FMV markup (100%/150%) is a SEPARATE fallback level** that should be inserted between the current Levels 4 and 6, producing a **7-level hierarchy** or a **branching tree** rather than a strict linear chain.
6. **A CRITICAL DISCREPANCY exists** between the formal RAMO 2-91 percentages (100%/150%) and current practitioner-reported RDO practice (20%). Both are documented but from different source types (formal issuance vs. practitioner observation).

## Additional Claims Verification

### Claim 1: RAMO 2-91 LGU FMV Percentages — Where Does It Fit?

**The question:** The prior analysis mentions "LGU FMV + 100% (residential/general) or LGU FMV + 150% (commercial/industrial/fishpond)" from RAMO 2-91. Where does this fit in the hierarchy?

| # | Source | Type | Finding | Verdict |
|---|--------|------|---------|---------|
| 1 | [RAMO 2-91 full text (SC E-Library)](https://elibrary.judiciary.gov.ph/thebookshelf/showdocs/10/49658) | Primary BIR issuance | **Exact text:** "When the zonal value of land has NOT been established, there shall be added to the market value per latest tax declaration ONE HUNDRED PERCENT (100%) thereof. PROVIDED, that if the property is classified as commercial, industrial and agricultural devoted to fishpond/prawn farm, ONE HUNDRED FIFTY PERCENT (150%) shall be added thereto." | **CONFIRMS** 100%/150% figures as formal rule. |
| 2 | [DOF DO — RDO 81 Cebu City North](http://macrealtyservices.com/wp-content/uploads/2014/07/RDO81-Cebu-CT-North.pdf) | Primary DOF DO footnote | "In the absence of zonal valuation, property shall be valued pursuant to RAMO 2-91." | **CONFIRMS** RAMO 2-91 as the fallback authority. |
| 3 | [DOF DO — RDO 83 Talisay/Minglanilla](http://macrealtyservices.com/wp-content/uploads/2014/07/RDO-83-Talisay-CT-Minglanilla.pdf) | Primary DOF DO footnote | Same RAMO 2-91 reference. | **CONFIRMS** |
| 4 | [Respicio & Co. — Basis for Zonal Valuation](https://www.respicio.ph/commentaries/basis-for-zonal-valuation-of-properties-in-the-philippines) | Legal commentary | "Where no zonal value exists (typically rural barangays), RDO uses the LGU FMV plus 20% or adopts the nearest comparable zone." | **CONTRADICTS** the 100%/150% figure — cites **20%** as current practice. |

**Analysis of the discrepancy:**

| Factor | RAMO 2-91 (100%/150%) | Respicio (20%) |
|--------|----------------------|----------------|
| **Source type** | Formal BIR administrative issuance (1991) | Legal commentary (2025) |
| **Legal authority** | Published in SC E-Library, referenced in DOF DOs | Practitioner observation, no formal issuance cited |
| **Date** | February 18, 1991 | Published August 2025 |
| **Context** | Formula when NO zonal value exists at all | Current RDO operational practice |

**Most likely explanation:** RAMO 2-91 was issued in 1991 when LGU tax declaration market values were significantly understated relative to actual FMV. The 100% markup was designed to approximate real market value. Over 35 years, many LGUs have updated their Schedules of Market Values closer to actual FMV, making the 100% markup excessive. Some RDOs may have informally adopted a lower markup (20%) — but this is not codified in any identified BIR issuance.

**Where it fits in the hierarchy:** RAMO 2-91 is a **SEPARATE level** that applies when:
- The entire area has NO zonal value for ANY classification (not just a specific classification gap)
- All prior fallback levels (exact match, same-barangay fallback, barangay catch-all, adjacent barangay, institutional fallback) have been exhausted
- The DOF DO footnotes explicitly state: "In the absence of zonal valuation, property shall be valued pursuant to RAMO 2-91"

It is **NOT the same as the institutional fallback** (Level 5). The institutional fallback is a classification-specific rule within an area that HAS zonal values (just not for X). RAMO 2-91 applies when the area has NO zonal values at all.

**RECOMMENDATION:** Insert RAMO 2-91 as **Level 5.5** (between current Levels 5 and 6), or restructure the hierarchy as a tree:

```
Level 1: Exact match
Level 2: Same street, different vicinity (same barangay)
Level 3: Barangay catch-all ("ALL OTHER STREETS/LOTS")
Level 4: Adjacent barangay, same classification
Level 5: Institutional → nearest commercial (X → CR)
Level 5A: RAMO 2-91 (LGU FMV + 100%/150%) — applies when NO zonal value exists at all
Level 6: NULL — advise RDO inquiry
```

**However, the engine should NOT compute the RAMO 2-91 markup.** Per CTA rulings (Emiliano, Gamboa), computing a value that does not exist in the published schedule crosses into territory where the BIR itself must follow Section 6(F) procedures. The engine should instead return: "No published zonal value exists. Per RAMO 2-91 and DOF DO footnotes, the RDO will apply the appropriate valuation formula."

**CLAIM 1 VERDICT: 100%/150% CONFIRMED as formal RAMO 2-91 text. 20% is SINGLE-SOURCE (Respicio only, no formal issuance). They are NOT the same rule — likely a formal vs. informal practice divergence. RAMO 2-91 is a separate level, not the same as the institutional fallback.**

## Structural Analysis: Proposed vs. Corrected Hierarchy

### Proposed Hierarchy (from the question)

```
Level 1: Exact Match
Level 2: Same Street, Different Vicinity
Level 3: Barangay Catch-All
Level 4: Adjacent Barangay
Level 5: Institutional → Commercial
Level 6: NULL
```

### Issues Identified

1. **Missing level:** RAMO 2-91 (LGU FMV + markup) is not in the proposed 6-level hierarchy, but it is a formally prescribed fallback (referenced in DOF DO footnotes: "In the absence of zonal valuation, property shall be valued pursuant to RAMO 2-91").

2. **Level 5 is classification-specific, not sequential:** The institutional → commercial fallback applies ONLY to X-classified properties. It should be a branch point, not a sequential level in the chain.

3. **Level 4 has two sub-rules:** Adjacent barangay for a specific classification gap (Rule 1) vs. adjacent barangay for a data-empty barangay (Rule 2). In practice, Rule 2 is relevant only during schedule creation, not during runtime lookup.

### Corrected Hierarchy (Recommended)

```
Level 1: Exact Match
         Street + vicinity + classification → direct value
         Authority: Section 6(E) NIRC; RMC 115-2020

Level 2: Same-Barangay Street Fallback
         Street not in schedule → "ALL OTHER STREETS" / nearest street of similar conditions
         Authority: DOF DO standard footnotes (Rule 3)

Level 3: Same-Barangay Classification Fallback
         Classification not listed for matched street → same classification from
         other street in same barangay of similar conditions
         Authority: DOF DO standard footnotes (Rule 3, variant)

Level 4: Adjacent Barangay
         No value for classification in entire barangay → same classification
         from adjacent barangay of similar conditions
         Authority: DOF DO standard footnotes (Rules 1 and 2)

Level 5: Institutional → Commercial (BRANCH — X classification only)
         X code with no X value → nearest commercial (CR) value in same
         barangay and street
         Authority: DOF DO standard footnotes (X classification provision)

Level 5A: RAMO 2-91 LGU FMV Markup (NOT COMPUTED BY ENGINE)
          No zonal value exists at all for the area →
          Tax declaration FMV + 100% (residential/general) or + 150%
          (commercial/industrial/fishpond)
          Authority: RAMO 2-91; DOF DO footnotes ("pursuant to RAMO 2-91")
          Engine action: Return reference to RAMO 2-91, do NOT compute value.

Level 6: NULL (No Match)
         No value found through any fallback → return no value
         Advise user to submit inquiry to RDO per Section 6(F) procedures
         Authority: CTA Emiliano (EB 1103, 2015); CTA Gamboa (9720, 2020)
```

**Key differences from the proposed hierarchy:**
1. Levels 2 and 3 distinguish street-gap vs. classification-gap scenarios (both within the same barangay).
2. Level 5 is explicitly marked as a branch that only applies to X-classified properties.
3. Level 5A (RAMO 2-91) is explicitly added with a note that the engine should NOT compute the value.
4. Authority attributions are corrected to DOF DO footnotes throughout.

## Critical Corrections Required for Engine Implementation

### Correction 1: Authority Attribution
Replace "RMO 31-2019" citations at Levels 2-4 with "DOF Department Order standard footnotes." RMO 31-2019 governs the process; DOF DOs contain the actual rules.

### Correction 2: RAMO 2-91 Is a Separate Level
Do not conflate the RAMO 2-91 LGU FMV markup with the institutional → commercial fallback. They address different scenarios:
- Institutional fallback: X code has no value, but the barangay HAS other zonal values
- RAMO 2-91: The entire area has NO zonal values at all

### Correction 3: Engine Must Never Compute Fallback Values
Per CTA rulings (Emiliano, Gamboa), Levels 5A and 6 should return informational results only — not computed values. The engine should cite the applicable rule and advise the user to work with their RDO.

### Correction 4: Institutional Fallback Scope Is "Same Barangay AND Street"
The DOF DO text says "within the same barangay and street," not just "within the same barangay." This is more restrictive than the proposed hierarchy suggests.

### Correction 5: CR Universality Has Limits
While CR is present in all sampled RDOs, purely agricultural barangays may lack CR entries entirely. The institutional fallback to CR could fail in rural provincial areas. The engine should handle this gracefully.
