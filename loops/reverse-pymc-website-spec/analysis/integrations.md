# Integrations — Third-Party Services Across Both Repos

## Overview

This analysis catalogs every third-party service integration across both the Lektor production site (`pymc-labs-website-source`) and the Next.js rebranded site (`pymc-rebranded-website`). Integrations are grouped by category with implementation details, configuration, and security concerns.

**Total unique third-party services identified: 17**
- Lektor site: 8 services
- Next.js site: 14 services
- Shared across both: 5 services (Mailchimp, Wise.com, YouTube, Font Awesome, social link platforms)

## 2. Email & Newsletter Services

### Mailchimp

| Property | Lektor | Next.js |
|----------|--------|---------|
| **Status** | Active | Active |
| **Account** | `us10.list-manage.com` (Thomas Wiecki's account) | Same account |
| **List ID** | `ffcf543278f48b79571b62010` | Same |
| **Form ID** | `cdca7f3ebb` | Same |
| **Newsletter** | HTML form in `templates/newsletter.html` | JSONP subscription in `components/layout/Footer.jsx:63` and `components/blog/Newsletter.jsx:23` |
| **Contact form** | Embedded iframe in `templates/contact.html` (`us10.list-manage.com` hosted page) | Not used for contact (Strapi handles contact forms) |

**Lektor implementation details**:
- Newsletter: Standard HTML `<form>` posting to Mailchimp action URL
- Contact: Full iframe embed of a Mailchimp-hosted form page — not a native form

**Next.js implementation details**:
- JSONP-based subscription via dynamic `<script>` tag injection
- URL: `https://twiecki.us10.list-manage.com/subscribe/post-json?u=ffcf543278f48b79571b62010&id=cdca7f3ebb&f_id=003f33e2f0`
- Parameters: `EMAIL`, `subscribe=Subscribe`, dynamic callback function name
- Duplicate implementation in Footer and blog Newsletter components (identical logic, copy-pasted)

**Security concerns**:
- Mailchimp list ID and form credentials hardcoded in client-side JS (both sites)
- JSONP is deprecated — vulnerable to script injection, incompatible with Content Security Policy
- No server-side proxy — credentials are directly exposed to the browser
- No error handling beyond `console.log` in the Next.js JSONP callbacks

### Icon Library Duplication (Next.js)

Three icon libraries are loaded simultaneously:
1. **Font Awesome 6.7.2** — CDN `<link>` in `app/layout.js`
2. **`lucide-react`** — npm package, tree-shakeable
3. **`react-icons`** — npm package, multiple icon packs (FontAwesome, Grommets, Ionicons, Bootstrap)

This results in redundant icon assets and increased bundle size. Should consolidate to a single library.
