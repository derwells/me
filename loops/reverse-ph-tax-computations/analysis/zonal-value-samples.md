## Legal Basis

- **NIRC Section 6(E)** (as amended by TRAIN / RA 10963) — authorizes the CIR to divide the Philippines into zones and determine fair market value per zone
- **EO 45, series of 1998** — mandates periodic review every 3–5 years
- **RMO No. 31-2019** — operative procedural order for establishing/revising zonal values; supersedes RMO 41-2010; defines Annex B (classification codes) and Annex C (table format)
- **RA 12001 / RPVARA (June 13, 2024)** — transfers authority from BIR CIR to LGU assessors under DOF/BLGF approval (transition period ending ~mid-2026); BIR zonal values remain in force until BLGF-approved SMVs replace them

**Approval chain:** STCRPV (RDO-chaired) → TCRPV (regional) → DOF Secretary signs a Department Order → 15 days after publication → effective

---

## Publication Format

Zonal values are published per RDO as:
- **Excel workbooks** (downloadable as ZIP from bir.gov.ph)
- **PDFs** for some older RDOs

Each workbook contains:
1. **NOTICE sheet** — guide to contents, how to navigate
2. **Revision schedule sheet** — all past revisions with effectivity dates (e.g., "6th Revision – DO 022-2017", "7th Revision – DO 022-2021")
3. **Zonal value sheets** — one or more per property classification or geographic grouping

### Official download path
```
bir.gov.ph → Zonal Values → Select Revenue Region → Select RDO → Download ZIP
```

---

## Geographic Hierarchy

Properties are organized by:
1. Revenue Region (19 nationwide; Metro Manila split across 7A, 7B, 8, 8A, 9)
2. Revenue District Office (127+ RDOs; each covers one or more cities/municipalities)
3. City / Municipality
4. Barangay
5. Street / Subdivision name
6. Vicinity descriptor (e.g., "Ayala Avenue — EDSA to Gil Puyat")

**Key resolution rule:** A single street often has multiple entries — one per classification (e.g., Shaw Blvd in Mandaluyong has both RR ₱120,000 and CR ₱150,000 because frontage lots are commercial and interior lots are residential).

## Classification Code System (Annex B, RMO 31-2019)

| Code | Classification | Definition |
|------|---------------|------------|
| RR | Residential Regular | Land principally devoted to habitation |
| CR | Commercial Regular | Land devoted principally to commercial use and profit |
| I | Industrial | Land devoted principally to industry |
| RC | Residential Condominium | Condo units for residential use |
| CC | Commercial Condominium | Condo units/office space for commercial use |
| PS | Parking Slot | Parking spaces — typically 60% of unit value |
| CL | Cemetery Lot | Cemetery/memorial park lots |
| GL | Government Land | Land owned/used by government |
| GP | General Purpose | Raw/undeveloped land, minimum 5,000 sqm |
| X | Institutional | Schools, churches, hospitals |
| APD | Area for Priority Development | Designated priority zones |
| A1–A50 | Agricultural sub-codes | 50+ categories (see below) |

**Agricultural sub-codes (selected):**
- A1: Riceland Irrigated
- A2: Riceland Unirrigated
- A4: Coco Land
- A6: Fishpond
- A14: Banana (Local/Export)
- A24: Mangrove
- A31: Mineral Land
- A36: Forest/Timber Land
- A50: Other Agricultural Lands

## Annex C Table Format (Standard Column Structure)

| Column | Content |
|--------|---------|
| Street / Subdivision / Barangay | Name |
| Vicinity | Further descriptor (street segments, etc.) |
| Classification Code | RR, CR, RC, CC, I, A1–A50, GP, CL, GL |
| ZV / SQ.M. | Zonal value in PHP per square meter |

For condominiums: separate sheet with building name, tower/floor range, RC (residential), CC (commercial), PS (parking) values.

## Update Frequency

**Formal mandate:** Every 3–5 years (EO 45, RMO 31-2019)

**Reality:** Highly uneven. As of 2024, DOF found:
- 38% of BIR zonal value schedules outdated
- 60% of LGU Schedule of Market Values outdated
- Of 227 LGUs checked, 137 had not revised in 3 years

**Typical revision gap by RDO example:**
- East Makati: approx. every 3–5 years (7 revisions since 1987)
- BGC Taguig: DO 040-2019 → DO 005-2024 (5-year gap)
- Mandaluyong: DO 059-2022

**RPVARA impact:** RA 12001 mandates LGU SMV updates every 3 years with penalties; BIR values remain operative through ~mid-2026 transition.

## Official Lookup Process (Step-by-Step)

1. **Identify RDO jurisdiction** — based on property's physical address (not owner's TIN)
2. **Download current schedule** — bir.gov.ph → Revenue Region → RDO → ZIP → latest revision
3. **Locate in workbook** — barangay sheet → street name → vicinity descriptor → classification code
4. **Read ZV/sq.m.** — multiply by property area for total zonal value
5. **Apply highest-of-three** — max(selling price, LGU FMV, zonal value × area) = tax base
6. **For unlisted properties** — request Certification of Zonal Value from RDO Zonal Valuation Section (bring technical description, tax declaration, TCT/CCT)

---

## Resolution Logic for Edge Cases

**Rule 1 — Street not listed:** Use the same classification's value from another street in the same barangay.

**Rule 2 — Predominant use rule:** If property classification differs from actual use, apply the predominant classification used by other properties in the same zone.

**Rule 3 — Special use (schools, hospitals, churches):** If unlisted, apply nearest commercial value within the same barangay.

**Rule 4 — No barangay value at all:** Use LGU FMV + 20%, or nearest comparable zone's value (RAMO 2-91 fallback).

**Rule 5 — Boundary disputes:** Written inquiry to RDO Zonal Valuation Section; escalation to BIR's Valuation and Classification Division at national office.

**Rule 6 — Newly reclassified land:** RDO issues Zonal Classification Ruling pending next revision cycle.

## Computational Implications for Automation

### What an automation engine needs:
1. **RDO jurisdiction lookup** — map property address → RDO number (requires address parsing)
2. **Zonal value data ingestion** — parse Excel workbooks from 127+ RDOs (heterogeneous formats)
3. **Property matching** — match barangay + street + vicinity descriptor → row in workbook
4. **Classification determination** — RR vs CR vs RC vs CC vs I vs A-codes (may require judgment on predominant use)
5. **Fallback rule engine** — chain of fallback rules when exact match fails
6. **Revision tracking** — track effectivity dates; flag stale schedules; RPVARA transition monitoring

### Key challenges for automation:
- **No API exists** — BIR publishes only downloadable ZIP/Excel files; no structured API
- **Format inconsistency** — each RDO's workbook may have slightly different column layouts
- **Image-based PDFs** — some RDOs (e.g., Taguig post-2024) publish image-only PDFs requiring OCR
- **Vicinity descriptor ambiguity** — "Ayala Avenue from EDSA to Gil Puyat" requires geocoding to match
- **Classification judgment** — whether a property is RR or CR within a zone sometimes requires local knowledge
- **Update tracking** — 127 RDOs update independently; no central revision notification system
- **RPVARA transition** — BIR → BLGF authority shift creates dual-data-source period through ~2026
